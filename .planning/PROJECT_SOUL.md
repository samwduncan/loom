# Loom V2: The Project Soul (Archival Memory)

*This document is the persistent conceptual core of Loom. It exists to anchor Claude when implementation details threaten to drown out the overarching vision.*

## The North Star
Loom is not a chat app; it is a **Construction Site for Software**. It must feel as precise as a surgical laser and as warm as an old library. 

## The Non-Negotiable "Feel" (The Sounding Board Checklist)
When debating an idea with Gemini, use these criteria to judge if a proposal is "Loom-worthy":
1. **Mathematical Precision:** Is the scroll logic absolute? Is there jitter? If yes, the solution is rejected.
2. **Intentional Negative Space:** Does the UI breathe? We prefer a centered, readable column over a screen cluttered with buttons.
3. **Tactile Physics:** Does it feel like a physical object? (Springs > Easing).
4. **Deep Charcoal & Dusty Rose:** No generic grays. The palette is the brand.

## The Architectural Anchor
- **Frontend:** Decoupled, ground-up React 18.
- **Backend:** Headless CloudCLI server.
- **State:** 4-Store Zustand isolation.
- **Rendering:** rAF Token Buffer (The "Anti-Jitter" Engine).
