   - Define an `INITIAL_TIMELINE_STATE` constant with all data fields (sessions, activeSessionId, activeProviderId). Use it in both the store creation and the `reset()` action to ensure deterministic state during HMR and test resets.

   - Actions (all use Immer draft mutation syntax):
     - `addSession(session: Session)`: push to sessions array
     - `removeSession(sessionId: string)`: filter out session
     - `setActiveSession(sessionId: string | null)`: set activeSessionId
     - `addMessage(sessionId: string, message: Message)`: find session, push message
     - `updateMessage(sessionId: string, messageId: string, updates: Partial<Message>)`: find session, find message, Object.assign draft
     - `clearSession(sessionId: string)`: find session, set messages to []
     - `updateSessionTitle(sessionId: string, title: string)`: find session, set title
     - `reset()`: restore `INITIAL_TIMELINE_STATE` (empty sessions array, null activeSessionId, 'claude' activeProviderId)
   - Persist config:
     - `name: 'loom-timeline'`
     - `version: 1`
     - `partialize`: whitelist-only — persist `sessions` mapped to metadata-only (id, title, providerId, createdAt, updatedAt, metadata — NO messages array), `activeSessionId`, `activeProviderId`
     - `migrate`: skeleton function that returns persistedState as-is for version 1
     - `merge`: custom merge function that ensures rehydrated sessions get empty `messages: []` arrays (since messages are never persisted)
