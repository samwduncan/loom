    Create `src/src/lib/tool-registry.ts` as a pure TypeScript module (no React imports in main module, only type-level ComponentType import):

    **ToolConfig interface:**
    - `displayName: string`
    - `icon: ComponentType` (React.ComponentType -- M1 icons are tiny components returning spans with Unicode: Bash=`▶`, Read=`📄`, Edit=`✏️`, Write=`📝`, Glob=`🔍`, Grep=`🔎`, default=`⚙️`)
    - `getChipLabel: (input: Record<string, unknown>) => string`

    - `input: Record<string, unknown>`
    - `output: string | null`
    - `isError: boolean`
    - `status: ToolCallStatus`

    **Registry internals:**
    - `const registry = new Map<string, ToolConfig>()`
    - `registerTool(toolName, config)` stores in Map
    - `getToolConfig(toolName)` returns Map entry or spreads DEFAULT_CONFIG with overridden displayName
    - `getRegisteredToolNames()` returns `[...registry.keys()]`

    **Chip label extraction helpers (private):**
    - `truncatePath(path: string, maxLen = 40)`: If path > maxLen, return `'...' + path.slice(-(maxLen - 3))`
    - `truncateCommand(cmd: string, maxLen = 40)`: If cmd > maxLen, return `cmd.slice(0, maxLen - 3) + '...'`
    - `defaultChipLabel(input)`: `JSON.stringify(input)` truncated to 40 chars

    **DefaultToolCard:** A simple component that renders tool input (JSON-formatted key field) and output (if present). Error output shown with inline text color using `var(--status-error)`. This is the shared expanded card for all M1 tools -- defined in tool-registry.ts as a minimal fallback. The full ToolCard component (Task 2) provides the styled version.

    **Registration:** Call `registerTool()` for all 6 tools at module scope (self-registering on import). Each tool has its own `getChipLabel` function that extracts the right input field.

    **IMPORTANT:** The icon components and DefaultToolCard component use JSX, so the file must be `.tsx` OR the icon/card components can be defined in a separate file. Recommendation: Keep tool-registry.ts as pure TypeScript with types and Map logic. Define the icon components and DefaultToolCard inline as arrow functions returning createElement calls (avoiding JSX requirement), OR rename to tool-registry.tsx. The test file references suggest .ts -- use createElement for the few component returns to keep the .ts extension.
