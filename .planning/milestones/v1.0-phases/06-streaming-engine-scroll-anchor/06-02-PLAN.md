<objective>
Create the scroll anchoring system that pins the chat scroll container to the bottom during streaming and disengages cleanly when the user scrolls up, with a floating pill for re-engagement.

Purpose: Chat UX depends on auto-scroll being invisible during streaming and instantly responsive to user intent. The IntersectionObserver sentinel approach is passive (zero CPU when not firing) and avoids the scroll event debouncing complexity.

Output: useScrollAnchor hook, ScrollToBottomPill component, and tests for both.
</objective>

    Create `src/src/hooks/useScrollAnchor.ts` implementing the IntersectionObserver scroll sentinel pattern.

    **Hook signature:**
    ```typescript
    export interface UseScrollAnchorReturn {
      sentinelRef: React.RefObject<HTMLDivElement | null>;
      isAtBottom: boolean;
      showPill: boolean;
      scrollToBottom: () => void;
    }
    export function useScrollAnchor(
      scrollContainerRef: React.RefObject<HTMLElement | null>
    ): UseScrollAnchorReturn
    ```

    **Implementation:**

    1. `sentinelRef = useRef<HTMLDivElement>(null)` -- zero-height div at bottom of scroll content
    2. `isAtBottom` state (useState, not ref -- drives pill visibility which IS a React render concern)
    3. `userScrolledRef = useRef(false)` -- guards against observer oscillation during rAF-driven auto-scroll
    4. Read `isStreaming` from `useStreamStore(state => state.isStreaming)`

    **IntersectionObserver setup (useEffect):**
    - Create observer with `{ root: scrollContainerRef.current, threshold: 0 }`
    - Observe `sentinelRef.current`
    - Callback: `([entry]) => { if (entry) setIsAtBottom(entry.isIntersecting) }`
    - Cleanup: `observer.disconnect()`

    **Anti-oscillation guard (Pitfall 3 from research):**
    The problem: during rAF auto-scroll, content grows -> observer fires "not intersecting" briefly -> auto-scroll catches up -> observer fires "intersecting". This makes the pill flash.

    Solution: Track whether we're driving the scroll programmatically via `isAutoScrollingRef`. When the rAF auto-scroll loop is active, ignore "not intersecting" transitions. Only treat "not intersecting" as a real user scroll when `isAutoScrollingRef.current === false`.

    In the observer callback:
    ```typescript
    ([entry]) => {
      if (!entry) return;
      if (entry.isIntersecting) {
        setIsAtBottom(true);
      } else if (!isAutoScrollingRef.current) {
        // Only disengage if we're NOT driving the scroll ourselves
        setIsAtBottom(false);
      }
    }
    ```

    **Auto-scroll rAF loop (useEffect):**
    - Guard: only runs when `isStreaming && isAtBottom`
    - Sets `isAutoScrollingRef.current = true`
    - rAF loop calls `sentinelRef.current?.scrollIntoView({ block: 'end' })`
    - Cleanup: `cancelAnimationFrame`, sets `isAutoScrollingRef.current = false`

    **New stream re-engagement (useEffect):**
    - Watch `isStreaming`. When it transitions from false to true (track previous value with a ref): `setIsAtBottom(true)` -- re-engage auto-scroll for new stream

    **scrollToBottom callback:**
    - `sentinelRef.current?.scrollIntoView({ behavior: 'smooth', block: 'end' })`
    - `setIsAtBottom(true)` -- re-engage auto-scroll

    **showPill derived:** `!isAtBottom && isStreaming` -- only show pill during active streaming when user has scrolled away

    Create `src/src/hooks/useScrollAnchor.test.ts` with comprehensive tests. Mock `IntersectionObserver` globally:
    ```typescript
    let observerCallback: IntersectionObserverCallback;
    const mockObserve = vi.fn();
    const mockDisconnect = vi.fn();
    vi.stubGlobal('IntersectionObserver', vi.fn().mockImplementation((cb) => {
      observerCallback = cb;
      return { observe: mockObserve, disconnect: mockDisconnect };
    }));
    ```

    Trigger observer events by calling `observerCallback([{ isIntersecting: true/false }], observer)` directly.

    Mock `requestAnimationFrame` with `vi.stubGlobal` for deterministic auto-scroll testing.
