  <action>
    Create `src/src/hooks/useStreamBuffer.ts` implementing the useRef + rAF token accumulation pattern:

    **Hook signature:**
    ```typescript
    export interface UseStreamBufferOptions {
      textNodeRef: React.RefObject<HTMLElement | null>;
      onFlush: (text: string) => void;
      onDisconnect?: () => void;
    }
    export function useStreamBuffer(options: UseStreamBufferOptions): { getText: () => string }
    ```

    **Implementation details:**
    1. `bufferRef = useRef('')` accumulates all tokens as string concatenation
    2. `isActiveRef = useRef(false)` guards rAF loop lifecycle
    3. `rafIdRef = useRef<number>(0)` stores current rAF handle for cancellation
    4. `paint` callback: reads `bufferRef.current`, writes to `textNodeRef.current.textContent`, reschedules via `requestAnimationFrame(paint)` if `isActiveRef.current` is true
    5. `onToken` callback: `bufferRef.current += token` -- no rAF scheduling, no state update, just string append
    6. `useEffect` subscribes to `wsClient.subscribeContent(onToken)`, starts rAF loop, returns cleanup that sets `isActiveRef = false`, cancels rAF, unsubscribes
    7. Watch `isStreaming` from stream store via selector. When `isStreaming` transitions from true to false AND `bufferRef.current` has content: call `onFlush(bufferRef.current)`
    8. Watch `wsClient.getState()` or use a separate effect to detect disconnect. When disconnected during active buffering, stop rAF loop and call `onDisconnect?.()`
    9. `getText()` return function exposes `bufferRef.current` for parent component reads

    **Critical:** The rAF loop does NOT schedule per token. It runs continuously while active, reading whatever has accumulated in the buffer since the last frame. Backlog replay from `subscribeContent()` is synchronous -- all replayed tokens land in `bufferRef.current` before the first rAF frame fires.

    **Disconnect handling:** Check `wsClient.getIsStreamActive()` in the paint loop. If stream is no longer active AND `wsClient.getState() === 'disconnected'`, stop the loop but do NOT clear the buffer (partial text stays visible per CONTEXT.md decision).

    Create `src/src/hooks/useStreamBuffer.test.ts` with comprehensive tests. Mock `wsClient` via `vi.mock('@/lib/websocket-client')`. Mock `requestAnimationFrame` to invoke callbacks synchronously for deterministic testing. Use `renderHook` from `@testing-library/react` for hook testing.

    For the isStreaming detection, subscribe to `useStreamStore(state => state.isStreaming)` inside the hook. Track previous value with a ref to detect the true->false transition. Call `onFlush` only on that transition, not on initial false.

      <span ref={textRef}></span>

    </div>
