Integrate file mentions into ChatComposer. This is the core wiring task.

**State additions:**
- Add `useState<FileMention[]>` for `mentions` (list of selected file mentions).
- Call `useFileMentions({ enabled: true, projectName })` to get picker state.

**onChange handler update:**
- After `setInput(e.target.value)`, call `detectAndOpen(e.target.value, e.target.selectionStart ?? e.target.value.length)` to check for @ triggers.

**onKeyDown handler update:**
When the mention picker is open, intercept these keys BEFORE the existing handleKeyDown logic:
- ArrowUp: call `moveUp()`, preventDefault
- ArrowDown: call `moveDown()`, preventDefault
- Enter: call `handleMentionSelect(selectCurrent())`, preventDefault (do NOT send the message)
- Escape: call `close()`, preventDefault (do NOT clear input)
- Tab: call `handleMentionSelect(selectCurrent())`, preventDefault

When the picker is NOT open, existing keyboard behavior is unchanged.

**handleMentionSelect function:**
When a file is selected (from picker Enter/click):
1. Add the FileMention to the `mentions` array (deduplicate by path).
2. Remove the `@query` text from the input. Find the @ position by scanning backward from cursor, remove from @ to cursor position.
3. Close the picker.
4. Refocus the textarea.

**MentionPicker rendering:**
Render `<MentionPicker>` inside the `.composer-pill` div, positioned above the textarea. Only render when `isOpen && results.length > 0` (or isLoading). Pass `onSelect={handleMentionSelect}` and `onClose={close}`.

**MentionChipRow rendering:**
Render `<MentionChipRow mentions={mentions} onRemove={handleRemoveMention} />` above the textarea, after ImagePreviewRow. `handleRemoveMention` filters the mention out of the array by path.

**handleSend update:**
When sending with file mentions:
1. Build the command text. If `mentions.length > 0`, prepend file context to the command:
   ```
   [Files referenced: src/foo.ts, src/bar.ts]

   {original message text}
   ```
   This ensures Claude reads the mentioned files. The fileMentions are also sent in `options.fileMentions` for future backend support.
2. After send, clear `mentions` to empty array (same as clearing input and images).

**hasContent update:**
Update the `hasContent` check: `input.trim() || attachments.length > 0 || mentions.length > 0`. A message with only file mentions (no text) should still be sendable -- but actually, that's a weird UX. Keep the existing behavior: require text OR images. Mentions alone don't enable send (they're context, not content).
