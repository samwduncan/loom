Create `src/src/hooks/useFileMentions.ts` with the following behavior:

The hook manages:
1. **File list fetching** -- On mount (when enabled), fetch files from `/api/projects/${projectName}/files` via apiFetch. Reuse the `FileEntry` type from command-palette/hooks/useCommandSearch.ts (import it, don't duplicate).
2. **Fuse.js index** -- Build a Fuse instance over fetched files with keys `['path', 'name']`, threshold 0.3 (same as command palette).
3. **@ trigger detection** -- Export a function `detectMentionQuery(text: string, cursorPos: number): string | null` that:
   - Scans backward from cursorPos to find the nearest unescaped `@`
   - Returns the query string after `@` (e.g., for "hello @App.t|" with cursor at end, returns "App.t")
   - Returns null if no `@` found or if `@` is preceded by a non-whitespace/non-start character (to avoid matching email addresses)
   - The `@` must be at position 0 OR preceded by whitespace
4. **Search** -- Export `searchFiles(query: string): FileMention[]` that uses the Fuse index, limited to 8 results, mapping FileEntry to FileMention.
5. **State** -- Track `isOpen: boolean`, `query: string`, `selectedIndex: number`. Expose `open()`, `close()`, `setQuery()`, `moveUp()`, `moveDown()`, `getSelected(): FileMention | null`.

The hook signature:
```typescript
export function useFileMentions(options: { enabled: boolean; projectName: string }): {
  isOpen: boolean;
  query: string;
  results: FileMention[];
  selectedIndex: number;
  detectAndOpen: (text: string, cursorPos: number) => void;
  close: () => void;
  moveUp: () => void;
  moveDown: () => void;
  selectCurrent: () => FileMention | null;
  isLoading: boolean;
}
```

`detectAndOpen` calls `detectMentionQuery`. If it returns a non-null query, open the picker and update results. If null, close the picker.

Write tests in `useFileMentions.test.ts`:
- `detectMentionQuery` returns query after @ at start of text
- `detectMentionQuery` returns query after @ preceded by space
- `detectMentionQuery` returns null when @ is inside a word (email-like)
- `detectMentionQuery` returns null when no @ present
- `detectMentionQuery` handles cursor in middle of text (only scans backward from cursor)
- Hook opens when @ detected, closes when @ removed
- Navigation: moveDown increments selectedIndex, moveUp decrements, wraps around
- selectCurrent returns the file at selectedIndex

Mock apiFetch for the file list in tests. Use `renderHook` from `@testing-library/react`.

Create `src/src/components/chat/composer/MentionPicker.tsx`:

A positioned popup that renders above the composer textarea. Props:

```typescript
interface MentionPickerProps {
  results: FileMention[];
  selectedIndex: number;
  isLoading: boolean;
  onSelect: (file: FileMention) => void;
  onClose: () => void;
}
```

Implementation:
- Render as an absolutely-positioned div anchored to the bottom of the popup (so it grows upward from the textarea). Use `bottom: 100%` positioning relative to the composer pill.
- Show a max of 8 results in a scrollable list (max-height ~240px).
- Each item shows the filename in primary text and the directory path in muted text (e.g., "App.tsx" bold, "src/components/" muted).
- The `selectedIndex` item gets `data-selected="true"` and a highlight background using `var(--surface-hover)`.
- Use `scrollIntoView({ block: 'nearest' })` on the selected item via a ref callback.
- If `isLoading` is true and results are empty, show a subtle "Loading files..." text.
- If results are empty and not loading, show "No files found" text.
- Clicking an item calls `onSelect(file)`.
- Use Tailwind classes + design tokens only (Constitution 3.1). No hardcoded colors.
- The popup should have a subtle border, raised surface background, and shadow matching the composer pill style.
