  <files>src/src/types/slash-command.ts, src/src/lib/slash-commands.ts, src/src/hooks/useSlashCommands.ts, src/src/hooks/useSlashCommands.test.ts</files>
  <behavior>
    - detectSlashQuery("/", 1) returns "" (empty query, show all)
    - detectSlashQuery("/cl", 3) returns "cl"
    - detectSlashQuery("hello /cl", 9) returns null (/ not at position 0)
    - detectSlashQuery("", 0) returns null
    - detectSlashQuery("hello", 5) returns null
    - SLASH_COMMANDS has exactly 4 entries: clear, help, compact, model
    - useSlashCommands: typing "/" at position 0 opens picker with all commands
    - useSlashCommands: typing "/cl" filters to /clear and /compact
    - useSlashCommands: moveUp/moveDown cycle through results
    - useSlashCommands: selectCurrent returns the selected command
    - useSlashCommands: close resets state
  </behavior>
  <action>
    1. Create `src/src/types/slash-command.ts`:
       ```typescript
       export interface SlashCommand {
         id: string;        // e.g., "clear"
         label: string;     // e.g., "/clear"
         description: string; // e.g., "Clear conversation"
         icon?: string;     // Optional lucide icon name
       }
       ```
       Note: No `action` callback on the type -- actions are handled by the consumer (ChatComposer) in Plan 02. The type is a pure data contract.

    2. Create `src/src/lib/slash-commands.ts`:
       ```typescript
       import type { SlashCommand } from '@/types/slash-command';

       export const SLASH_COMMANDS: SlashCommand[] = [
         { id: 'clear', label: '/clear', description: 'Clear conversation history' },
         { id: 'help', label: '/help', description: 'Show available commands' },
         { id: 'compact', label: '/compact', description: 'Compact conversation context' },
         { id: 'model', label: '/model', description: 'Switch AI model' },
       ];
       ```

    3. Create `src/src/hooks/useSlashCommands.ts` following the EXACT same pattern as `useFileMentions.ts`:
       - Export `detectSlashQuery(text: string, cursorPos: number): string | null` -- returns query after `/` ONLY if `/` is at position 0 of the text. The `/` must be the FIRST character (no preceding whitespace needed unlike @). If the text before cursor contains whitespace after the `/`, return null (user moved past the command).
       - Export `useSlashCommands()` hook returning: `{ isOpen, query, results, selectedIndex, detectAndOpen, close, moveUp, moveDown, selectCurrent }`.
       - No `isLoading` needed (commands are static, no fetch).
       - `results` are `SlashCommand[]` filtered by query matching against `id` field (simple `startsWith` or `includes` -- no fuse.js needed for 4 items).
       - `detectAndOpen(text, cursorPos)` calls `detectSlashQuery` and sets state accordingly.
       - `selectCurrent()` returns `SlashCommand | null`.
       - `moveUp`, `moveDown`, `close` follow the same wrap-around pattern as useFileMentions.

    4. Create test file `src/src/hooks/useSlashCommands.test.ts`:
       - Test `detectSlashQuery` pure function exhaustively (see behavior list above).
       - Test `useSlashCommands` hook via `renderHook` -- open/close, filtering, navigation, selection.
       - Follow the same test structure as `src/src/hooks/useFileMentions.test.ts`.
  </action>

  </verify>
