  <name>Task 1: Create useGitFileMap hook with directory aggregation</name>
  <files>src/src/hooks/useGitFileMap.ts, src/src/hooks/useGitFileMap.test.ts</files>
  <behavior>
    - Given an empty array, returns an empty Map
    - Given [{path: "src/foo.ts", status: "modified"}], map has "src/foo.ts" -> "modified"
    - Given [{path: "src/lib/bar.ts", status: "added"}], map has entries for "src/lib/bar.ts" -> "added", "src/lib" -> "added", "src" -> "added" (directory aggregation)
    - Directory with mixed children uses priority order: modified > added > deleted > untracked (i.e., if a directory has both a modified and an untracked child, the directory shows "modified")
    - Given [{path: "a/x.ts", status: "deleted"}, {path: "a/y.ts", status: "untracked"}], "a" -> "deleted" (deleted beats untracked)
    - Returns the same Map reference when input array is referentially stable (useMemo)
    - Handles files at root level (no directory segments) without error
  </behavior>
  <action>
    Create `src/src/hooks/useGitFileMap.ts`:
    - Export `useGitFileMap(files: GitFileChange[] | undefined): Map<string, GitFileStatus>`
    - Wraps computation in `useMemo` keyed on `files` reference
    - For each file, add an entry for the file path itself, then walk up the path splitting on "/" to add/update directory entries
    - Directory entries use priority: modified (highest) > added > deleted > untracked (lowest). Use a numeric priority map to compare.
    - Return `EMPTY_MAP` constant (stable reference) when files is undefined/empty

    Create tests in `src/src/hooks/useGitFileMap.test.ts`:
    - Test all behaviors listed above
    - Use `renderHook` from `@testing-library/react` to test the hook
    - Verify Map contents with `.get()` assertions
