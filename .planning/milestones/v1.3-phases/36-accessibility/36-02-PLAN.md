  <action>
    **useFocusRestore hook:**
    Create `src/src/components/a11y/useFocusRestore.ts`. A small hook that:
    - On mount (or when `isOpen` transitions to true), captures `document.activeElement` into a ref
    - On unmount (or when `isOpen` transitions to false), restores focus to the captured element via `requestAnimationFrame(() => ref.current?.focus())`
    - API: `useFocusRestore(isOpen: boolean)` — no return value needed
    - This is for components NOT using Radix (which handles focus restore itself). Primary consumers: ImageLightbox, any custom overlay.

    **TabBar panel focus on switch:**
    In TabBar.tsx, when the active tab changes (the `setActiveTab` handler or useEffect watching activeTab), after setting the new tab, focus the corresponding panel container: `document.getElementById('panel-' + newTabId)?.focus()`. The panel containers (ContentArea tabpanels) already have `role="tabpanel"` and `id` attributes. Add `tabIndex={-1}` to each tabpanel div if not already present (allows programmatic focus without adding to tab order). This is critical because the mount-once CSS show/hide pattern means focus can get stuck in a hidden panel.

    **ImageLightbox focus trap:**
    Check if `src/src/components/chat/view/ImageLightbox.tsx` exists. If it does:
    - Wire `useFocusRestore(isOpen)` into it
    - If it's NOT using Radix Dialog, add a basic focus trap: on open, focus the close button; on keydown Tab, constrain focus within the lightbox; on Escape, close
    - If it IS using Radix Dialog, no changes needed (Radix handles it)
    If ImageLightbox doesn't exist, skip this substep.

    **Tests (a11y-focus.test.tsx):**
    Create `src/src/tests/a11y-focus.test.tsx`:
    1. "useFocusRestore restores focus on close" — render a button + overlay toggle, open overlay, close it, verify button regains focus

    Mock stores as needed following existing test patterns (useUIStore, useStreamStore).

  <action>
    **LiveAnnouncer component:**
    Create `src/src/components/a11y/LiveAnnouncer.tsx`:
    - Renders a visually-hidden `<div role="status" aria-live="polite" aria-atomic="true" className="sr-only">` containing the current announcement text
    - Props: `message: string`
    - When `message` changes: clear current text, then after 100ms timeout set the new text (this clear-then-set ensures re-announcement of identical consecutive messages)
    - Use `useEffect` with cleanup for the timeout
    - Export as named export

    **useStreamAnnouncements hook:**
    Create within the same file (or as a separate hook in the a11y directory): `useStreamAnnouncements()` that:
    - Subscribes to `useStreamStore` selectors: `isStreaming`, `activeToolCalls`
    - Tracks previous tool call count via useRef
    - Returns a `message: string` that updates on:
      1. `isStreaming` becomes true -> "Assistant is responding"
      2. `isStreaming` becomes false (and was true) -> "Response complete"
      3. A new tool call completes (status changes to 'complete') -> "Tool {toolName} completed"
    - Does NOT announce every streaming text chunk (that would flood the screen reader)
    - Uses `useEffect` watching `isStreaming` and `activeToolCalls` length/status

    **ChatView integration:**
    In ChatView.tsx, import `LiveAnnouncer` and `useStreamAnnouncements`. Add near the top of the JSX (before the grid container):
    ```tsx
    const announcement = useStreamAnnouncements();
    // ... in JSX:
    <LiveAnnouncer message={announcement} />
    ```

    **AppShell integration (error announcements):**
    The ConnectionBanner already uses `role="alert"` which is assertive. No changes needed there. But add a global `LiveAnnouncer` in AppShell for any future general announcements — render it with an empty message for now as a mount point. This is optional — skip if it adds unnecessary complexity.

    2. "LiveAnnouncer re-announces when message changes" — render, change message prop, verify text updates after timeout
    3. "useStreamAnnouncements announces streaming start" — mock useStreamStore to return isStreaming:true, verify hook returns "Assistant is responding"
    4. "useStreamAnnouncements announces streaming end" — toggle isStreaming false, verify "Response complete"
    5. "useStreamAnnouncements announces tool completion" — add a tool call with status 'complete', verify announcement

    Use `vi.useFakeTimers()` for the 100ms timeout assertions. Mock `useStreamStore` following existing patterns.
