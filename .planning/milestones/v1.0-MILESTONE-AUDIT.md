## Critical Integration Gaps

### INT-01: initializeWebSocket() not called from main app (CRITICAL)

**Severity:** Critical — breaks 3 E2E flows
**From:** `src/src/main.tsx` / `src/src/App.tsx`
**To:** `src/src/lib/websocket-init.ts`

`initializeWebSocket()` is only called from `ProofOfLife.tsx` (the dev demo at `/dev/proof-of-life`). The production `/chat` route never initializes the WebSocket. This means:
- `wsClient` is never configured (no callbacks)
- `wsClient` is never connected (no auth bootstrap)
- `ChatComposer.handleSend()` calls `wsClient.send()` which returns `false`
- No streaming responses are received

**Affected:** STRM-01, STRM-02, STRM-03, NAV-02
**Fix:** Add `initializeWebSocket()` call to `App.tsx` (useEffect on mount) or `main.tsx` (before render)

### INT-02: SessionList click doesn't navigate (MEDIUM)

**Severity:** Medium — sidebar clicks don't trigger message loading
**From:** `src/src/components/sidebar/SessionList.tsx` line 48-52
**To:** `src/src/components/chat/view/ChatView.tsx`

`handleSessionClick` calls `setActiveSession(sessionId)` but does NOT call `navigate()`. ChatView reads `sessionId` from `useParams()` (URL), so the URL stays unchanged, ChatView's useEffect never detects a session change, and `useSessionSwitch.switchSession()` never fires.

**Affected:** NAV-01, NAV-02
**Fix:** Add `useNavigate()` and call `navigate(\`/chat/${sessionId}\`)` in `handleSessionClick`
