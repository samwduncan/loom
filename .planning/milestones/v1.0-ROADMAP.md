# Milestone v1.0: The Skeleton

**Status:** SHIPPED 2026-03-07
**Phases:** 1-10
**Total Plans:** 21

## Overview

Milestone 1 built the architectural skeleton for the entire Loom V2 frontend and proved it works with a real vertical slice: a streamed AI response with thinking blocks rendering in the browser. The build order followed a strict dependency chain — design tokens and enforcement first (preventing the foundation rot that killed V1), then state and streaming infrastructure, then the proof-of-life vertical slice, navigation to make it usable, and finally E2E integration wiring and cleanup.

## Milestone Summary

**Key Decisions:**
- React 19 used (Vite template ships 19, backwards compatible with 18 target)
- OKLCH color space with warm charcoal hue 32, dusty rose hue 20
- WebSocket callback injection pattern (configure method) instead of direct store imports
- Multiplexer as pure functions with zero store/React imports
- useRef + rAF DOM mutation for streaming (zero React re-renders)
- Segment array architecture for ActiveMessage (interleaved text + ToolChip)
- Callback ref + useState for IntersectionObserver sentinel attachment
- Module-level singleton for project name resolution
- Zustand persist with empty messages arrays (messages never persisted)
- Playwright E2E with predicate to distinguish app WS from Vite HMR WS

**Issues Resolved:**
- INT-01: WebSocket init not called from production route (Phase 9)
- INT-02: SessionList click didn't navigate (Phase 9)
- ToolCard.tsx orphaned export (Phase 10)
- Inline styles in tool-registry.ts (Phase 10)
- Stale ENF-01 traceability (Phase 10)
- Unused _resetProjectContextForTesting export (Phase 10)

**Issues Deferred:**
