If testing shows oklch strings are returned literally, add a fallback: parse to hex via a canvas 2d context trick or hardcode matching hex values. Document the approach in a code comment.

<task type="auto" tdd="true">
  <name>Task 2: Shell WebSocket client class and React hook</name>
  <files>src/src/lib/shell-ws-client.ts, src/src/hooks/useShellWebSocket.ts, src/src/hooks/useShellWebSocket.test.ts</files>
  <behavior>
    - connect() creates WebSocket to ws://{host}/shell?token={jwt} and transitions state to 'connecting'
    - On WS open, sends init message with projectPath, plain-shell mode, cols, rows -- state becomes 'connected'
    - On WS message with type 'output', calls onOutput callback with data string
    - On WS message with type 'auth_url', calls onAuthUrl callback with url and autoOpen
    - sendInput(data) sends {type:'input', data} over WS
    - sendResize(cols, rows) sends {type:'resize', cols, rows} over WS
    - disconnect() closes WS with code 1000, transitions to 'disconnected'
    - React strict mode: second useEffect mount does NOT create second WS connection (ref guard)
    - restart() disconnects then reconnects with same params
  </behavior>
  <action>
1. Create `src/src/lib/shell-ws-client.ts`:
   - Class `ShellWebSocketClient` (not a singleton -- each terminal panel gets its own instance).
   - Constructor takes no args. Connection params passed to `connect()`.
   - `connect(params: { token: string; projectPath: string; isPlainShell: boolean; cols: number; rows: number })`: Build URL as `${protocol}//${window.location.host}/shell?token=${token}`. Create WebSocket. On open, send init message. On message, parse JSON and dispatch to onOutput/onAuthUrl callbacks. On close/error, update state to disconnected.
   - `sendInput(data: string)`, `sendResize(cols: number, rows: number)`: Send typed JSON messages.
   - `disconnect()`: Close WS with code 1000 if open, null out ws reference.
   - State tracking: `state` getter returns ShellConnectionState. State change callback via `onStateChange`.
   - Callbacks: `onOutput: ((data: string) => void) | null`, `onAuthUrl: ((url: string, autoOpen: boolean) => void) | null`, `onStateChange: ((state: ShellConnectionState) => void) | null`.

2. Create `src/src/hooks/useShellWebSocket.ts`:
   - Hook signature: `useShellWebSocket(params: { projectPath: string; isPlainShell?: boolean })` returns `{ state: ShellConnectionState; sendInput, sendResize, connect, disconnect, restart, authUrl: { url: string; autoOpen: boolean } | null }`.
   - Creates `ShellWebSocketClient` instance in a ref (stable across renders).
   - `connect` function: calls `getToken()`, gets terminal cols/rows from caller, creates connection.
   - React strict mode guard: `useRef<boolean>(false)` -- if `initialized.current` is true on mount, skip. Set to true after first init. Reset to false on cleanup.
   - `authUrl` state: useState for latest auth URL from backend. Set on onAuthUrl callback. Cleared on dismiss.
   - Cleanup: disconnect WS on unmount.
   - Does NOT auto-connect on mount -- caller (TerminalView) connects after xterm.open() so cols/rows are known.

3. Create `src/src/hooks/useShellWebSocket.test.ts`:
   - Mock WebSocket globally (class with send/close/addEventListener stubs).
   - Mock `getToken` to return 'test-token'.
   - Test: connect creates WebSocket with correct URL including token.
   - Test: on WS open, init message sent with projectPath and plain-shell params.
   - Test: on WS message with type 'output', onOutput called.
   - Test: on WS message with type 'auth_url', authUrl state updates.
   - Test: sendInput sends correct JSON.
   - Test: sendResize sends correct JSON.
   - Test: disconnect closes WS with code 1000.
   - Test: strict mode guard -- second init call is no-op when initialized ref is true.
  </action>
