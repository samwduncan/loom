The cost comes later from the `result` message, so this provides partial data early.    - `const newStartIndex = (isInitialLoadRef.current || sessionSwitchRef.current) ? messages.length : prevMessageCountRef.current;`
   - After render: `useEffect(() => { prevMessageCountRef.current = messages.length; sessionSwitchRef.current = false; }, [messages.length, sessionId])`.

3. **Apply animation classes**:
   - In the messages map, wrap each message in its animation container (or apply directly to the existing wrapper):
   ```tsx
   {messages.map((msg, idx) => (

       <div className={cn(
         idx >= newStartIndex && 'motion-safe:animate-in motion-safe:fade-in motion-safe:slide-in-from-bottom-2 duration-200'

         {renderMessage(msg)}
   ```
