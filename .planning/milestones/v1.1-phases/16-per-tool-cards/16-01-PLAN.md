    - Export `parseAnsi(text: string): string` -- converts ANSI escape sequences to HTML spans with CSS classes
    - Track style state: `{ bold: boolean, underline: boolean, fg: number | null }`
    - Parse ESC[...m sequences (regex: `/\x1b\[([0-9;]*)m/g`)
    - Handle semicolon-separated codes in single sequence (e.g. `\x1b[1;31m` = bold + red)
    - SGR codes: 0=reset, 1=bold, 4=underline, 30-37=fg color, 90-97=bright fg
    - Emit `<span class="ansi-red ansi-bold">` etc. Close span on style change or reset.
    - HTML-escape the text content (`<`, `>`, `&`) BEFORE wrapping in spans
    - Return plain text unchanged if no ANSI sequences found

      - `.ansi-blue { color: var(--status-info); }`
      - `.ansi-magenta { color: var(--accent-primary); }`

    - Shared card body styles: `.tool-card-body { font-family: var(--font-mono); font-size: var(--text-sm); }`

    **5. Create `src/src/lib/diff-parser.ts`:**
    - Export `DiffLine` interface: `{ type: 'added' | 'removed' | 'context', content: string, oldLineNo: number | null, newLineNo: number | null }`
    - Export `computeDiff(oldStr: string, newStr: string, contextLines = 3): DiffLine[]`
    - Use `structuredPatch` from `diff` package
    - Handle edge cases: empty oldStr (all additions), empty newStr (all deletions)
    - Return empty array if both strings are identical

    **6. Create `src/src/lib/grep-parser.ts`:**
    - Export `GrepMatch`: `{ lineNo: number, content: string }`
    - Export `GrepFileGroup`: `{ filePath: string, matches: GrepMatch[] }`
    - Export `parseGrepOutput(output: string): GrepFileGroup[] | null`
    - Parse ripgrep format: `filepath:linenum:content` lines
    - Handle `--` separator lines between context groups
    - Group consecutive matches by filepath
    - Return `null` if no lines match expected pattern (triggers plain text fallback)
