**useAutoResize.ts** -- Custom hook using `useLayoutEffect` (NOT useEffect):
- Accepts `ref: React.RefObject<HTMLTextAreaElement | null>`, `value: string`, `maxHeight = 200`
- On every value change: set `el.style.height = '0px'`, read `el.scrollHeight`, clamp to `maxHeight`, set `el.style.height`, set `overflowY` to `auto` if over max, `hidden` if under
- Dependency array: `[ref, value, maxHeight]`
- Named export only

**useComposerState.ts** -- 5-state FSM via `useReducer`:
- States: `'idle' | 'sending' | 'active' | 'aborting'`
- Actions: `SEND`, `STREAM_STARTED`, `STOP`, `STREAM_ENDED`, `ABORT_TIMEOUT`
- Transitions per CONTEXT.md:
  - idle + SEND -> sending
  - sending + STREAM_STARTED -> active
  - active + STOP -> aborting
  - active + STREAM_ENDED -> idle
  - aborting + STREAM_ENDED -> idle
  - aborting + ABORT_TIMEOUT -> idle
  - All other combos -> no-op (return current state)
- Hook returns `{ state, dispatch, canSend: boolean, canStop: boolean }`
  - `canSend`: state === 'idle'
  - `canStop`: state === 'active'
- Wire `isStreaming` from stream store: `useEffect` that dispatches `STREAM_STARTED` when isStreaming flips true and state is 'sending', dispatches `STREAM_ENDED` when isStreaming flips false and state is 'active' or 'aborting'
- 200ms lockout for sending: after SEND dispatch, `setTimeout(() => {}, 200)` -- actually this is handled by the state machine itself (can't send again until idle)
- 5-second abort timeout: when entering 'aborting', start a timeout that dispatches `ABORT_TIMEOUT`. Clear on unmount and on `STREAM_ENDED`.
- Named exports: `useComposerState`, `ComposerState`, `ComposerAction`

**composer.css** -- Composer-specific styles using design tokens ONLY:
- `.composer-pill`: `rounded-xl`, shadow, `bg-surface-1`, padding, border, focus-within ring (`ring-primary/30`)
- `.composer-pill[data-streaming="true"]`: border shifts to `border-primary/40`
- `.composer-button-cell`: Grid-based crossfade container (`display: grid; grid-template-areas: "button"`)
- `.composer-button-cell > *`: `grid-area: button; transition: opacity var(--duration-fast) var(--ease-out)`
- `[data-visible="false"]`: `opacity: 0; pointer-events: none`
- `.composer-abort-spinner`: small CSS spinner (border-based, 16x16, `border-muted`, `border-t-primary`, `animation: spin 0.8s linear infinite`)
- `@keyframes spin { to { transform: rotate(360deg) } }`
- All durations use `var(--duration-fast)`, `var(--duration-normal)` etc. NO hardcoded values.

Key implementation details:
- Replace `<input>` with `<textarea>` using `useAutoResize` hook. `rows={1}`, `resize: none` CSS.
- Wire `useComposerState` FSM. Send button visible when idle + non-empty input. Stop button visible when active. Aborting shows CSS spinner (from composer.css).
- Send button: Lucide `Send` icon (import from `lucide-react`), `bg-primary text-surface-base`, disabled when state !== idle or input empty
- Stop button: Lucide `Square` icon, `bg-status-error text-foreground`, disabled when state !== active
- Both buttons in same grid cell via `composer-button-cell`, toggle visibility via `data-visible` attribute
- `handleSend`: Preserve ALL existing M1 logic (stub session creation, navigate, optimistic message, pendingTitleRef, wsClient.send) but:
  - Allow sending during streaming (message queuing): if `isStreaming && input.trim()`, send the message with same wsClient.send pattern, add optimistic user message with a `queued: true` flag in metadata, do NOT dispatch SEND (stay in current state)
  - When NOT streaming: dispatch SEND, then send via wsClient
- `handleStop`: Send `abort-session` (existing type), dispatch STOP
- Keyboard handling:
  - `onKeyDown` on textarea: Enter (no shift) -> send. Shift+Enter -> default (newline). Escape -> if input non-empty, clear input; if input empty, blur textarea.
  - Global `useEffect` for Cmd+./Ctrl+. shortcut: `keydown` listener on `document`, check `e.metaKey || e.ctrlKey` + `e.key === '.'`, only fire when streaming is active, call handleStop. Don't `preventDefault` when not streaming.
- Auto-focus: `useEffect` on sessionId change -> `textareaRef.current?.focus()`. After send -> `requestAnimationFrame(() => textareaRef.current?.focus())`.
- Scroll stability: Before auto-resize triggers, capture `scrollContainer.scrollTop` and `scrollContainer.scrollHeight`. After resize in a `useLayoutEffect`, calculate delta and adjust scrollTop. Get scroll container ref via `document.querySelector('[data-testid="message-list-scroll"]')` or pass via prop/context.
- `hasMessageSentRef`: `useRef(false)`, set true after first successful send. Passed to ComposerKeyboardHints.
- Outer wrapper: `max-w-3xl mx-auto w-full px-4 pb-4 pt-2` for centered column alignment
