**useImageAttachments.ts** -- Hook managing the full image attachment lifecycle:

State: `attachments: Array<{ id: string; file: File; previewUrl: string }>`

Methods:
- `addImages(files: File[])`: Validate each file:
  - Check `file.type.startsWith('image/')` -- reject non-images with `toast.error('Only images are supported')`
  - Check `file.size <= 5 * 1024 * 1024` -- reject with `toast.error('Image too large: X.XMB (max 5MB)')` (use Sonner, already imported)
  - Check `attachments.length < 5` -- reject with `toast.error('Maximum 5 images per message')`
  - Create preview: `URL.createObjectURL(file)`, generate id: `Math.random().toString(36).slice(2, 10)`
  - Add to state
- `removeImage(id: string)`: Revoke the object URL via `URL.revokeObjectURL(previewUrl)`, remove from state
- `clearAll()`: Revoke ALL preview URLs, reset state to empty array
- `getBase64ForSend()`: Returns `Promise<Array<{ data: string; name: string }>>` -- uses `FileReader.readAsDataURL` for each file. This is called at send time, NOT during preview. Pattern from RESEARCH.md.
- `addFromClipboard(e: React.ClipboardEvent)`: Extract `image/*` items from `e.clipboardData.items`, call `item.getAsFile()`, pass to `addImages`
- `addFromDrop(e: React.DragEvent)`: Extract files from `e.dataTransfer.files`, filter to images, pass to `addImages`

Cleanup: `useEffect` cleanup on unmount that revokes all remaining preview URLs.
