    - parseThinkingMarkdown("**bold**") returns HTML with `<strong>bold</strong>`
    - parseThinkingMarkdown("*italic*") returns HTML with `<em>italic</em>`
    - parseThinkingMarkdown("`code`") returns HTML with `<code class="thinking-code">code</code>`
    - parseThinkingMarkdown("[link](https://example.com)") returns HTML with `<a href="...">link</a>`
    - parseThinkingMarkdown("**bold** and `code`") handles mixed inline formatting
    - parseThinkingMarkdown("") returns empty string
    - parseThinkingMarkdown with `<script>` tags sanitizes them away (DOMPurify)
    - Code inside backticks is NOT processed for bold/italic (backticks extracted first)
    - No block-level elements produced (no headings, lists, code fences)
  </behavior>
  <action>
**Create `src/src/lib/thinking-markdown.ts`:**
- Export `parseThinkingMarkdown(text: string): string`
- Reuse regex patterns from `streaming-markdown.ts` `processInlineFormatting()` function but as standalone function
- Processing order: (1) escape HTML, (2) extract inline code spans to placeholders, (3) convert `**bold**`/`__bold__` to `<strong>`, (4) convert `*italic*`/`_italic_` to `<em>`, (5) convert `[text](url)` to `<a href target="_blank" rel="noopener noreferrer">`, (6) restore code placeholders as `<code class="thinking-code">`, (7) sanitize with DOMPurify using strict ALLOWED_TAGS: `['strong', 'em', 'code', 'a']` and ALLOWED_ATTR: `['href', 'target', 'rel', 'class']`
- The `thinking-code` CSS class will be added to ThinkingDisclosure's existing CSS: `background: var(--surface-overlay); border-radius: 2px; padding: 0 4px;` -- visually distinct from surrounding monospace
