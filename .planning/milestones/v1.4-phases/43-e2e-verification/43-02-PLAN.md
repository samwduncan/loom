Create three Playwright spec files:

**quick-settings.spec.ts** (E2E-09):
- Navigate to /chat
- Open the quick settings panel (inspect QuickSettingsPanel.tsx and Sidebar.tsx for the trigger — likely a gear icon or keyboard shortcut)
- Find a toggle switch (e.g., show thinking, show token usage, or any available toggle)

- Reload the page (`page.reload()`)
- Re-open quick settings panel
- Verify the toggle persisted in the new state
- Toggle it back to restore original state
- Add data-testid attributes to QuickSettingsPanel.tsx toggles if needed

**auto-collapse.spec.ts** (E2E-10):
- Navigate to an existing session with multiple conversation turns (or create multiple turns by sending several messages)
- This requires either: (a) a known session with many turns, or (b) sending 3-4 short messages to build up turns
- Practical approach: send 2-3 quick messages ("Say: 1", "Say: 2", "Say: 3"), waiting for each to complete
- Scroll to bottom, then scroll up past older turns
- Verify that turns near the top auto-collapse (look for collapsed state — reduced height, summary text, or a data-attribute like data-collapsed="true")
- If MessageList.tsx uses IntersectionObserver with a specific threshold, the test needs enough messages that some leave the viewport
- Mark test.slow() — multiple API calls needed
- If auto-collapse is not reliably testable in E2E (timing-dependent, viewport-dependent), write the test to verify the IntersectionObserver setup exists and fires, or skip gracefully with a documented reason

**navigate-guard.spec.ts** (E2E-11):
- Navigate to /chat, establish WebSocket connection
- Send a message to start streaming
- While streaming is active (before streaming-cursor disappears), attempt to trigger beforeunload
- Playwright approach: use `page.evaluate(() => window.dispatchEvent(new Event('beforeunload')))` to check if the handler is registered
- Better approach: use `page.on('dialog')` listener, then try `page.close()` or `page.goto('about:blank')` during streaming — the beforeunload should trigger a confirmation dialog
- Verify the dialog appears with the expected message text
- Accept the dialog to let the test complete cleanly
- Mark test.slow() — needs active streaming
