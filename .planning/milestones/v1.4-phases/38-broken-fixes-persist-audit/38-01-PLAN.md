```javascript
// Handle file mentions - read files and prepend content to prompt
if (options.fileMentions && options.fileMentions.length > 0) {
  const fs = require('fs');
  const path = require('path');
  const projectDir = options.projectPath || options.cwd || process.cwd();
  const fileContents = [];
  for (const filePath of options.fileMentions) {
    try {
      const absPath = path.isAbsolute(filePath) ? filePath : path.join(projectDir, filePath);
      const content = fs.readFileSync(absPath, 'utf-8');
      fileContents.push(`<file path="${filePath}">\n${content}\n</file>`);
    } catch (err) {
      fileContents.push(`<file path="${filePath}" error="Could not read: ${err.message}" />`);
    }
  }
  if (fileContents.length > 0) {
    finalCommand = `${fileContents.join('\n\n')}\n\n${finalCommand}`;
  }
}
```

**FIX-02: Search highlighting in assistant messages**

The bug: `AssistantMessage` receives `highlightText` prop and threads it to `ThinkingDisclosure`, but does NOT thread it to `MarkdownRenderer` (line 80). `MarkdownRenderer` doesn't even accept a `highlightText` prop.

The fix approach: Adding search highlighting INSIDE react-markdown is complex and fragile (would need a custom rehype plugin to walk the AST and wrap matches). Instead, use a simpler approach — when `highlightText` is provided and there's a search query, render content through `highlightText` INSTEAD of `MarkdownRenderer`. This trades rich markdown for visible search matches, which is the correct UX tradeoff during active search.

In `AssistantMessage.tsx`:
- When `highlightText` is provided, wrap the content in a `<div className="markdown-body text-foreground text-sm leading-relaxed whitespace-pre-wrap">` and render `highlightText(message.content)` instead of `<MarkdownRenderer>`. This shows plain text with highlights during search — acceptable since search is a temporary overlay state.
- When `highlightText` is NOT provided (normal display), render `<MarkdownRenderer>` as before.

```tsx
{highlightText ? (
  <div className="markdown-body text-foreground text-sm leading-relaxed whitespace-pre-wrap">
    {highlightText(message.content)}
  </div>
) : (
  <MarkdownRenderer content={message.content} />
)}
```
