    **SessionList.tsx changes:**

    1. Replace `useTimelineStore` session reading with `useMultiProjectSessions()`:
    ```
    const { projectGroups, isLoading: multiLoading, error: multiError, expandedProjects, toggleProject } = useMultiProjectSessions();
    ```

    2. Keep the timeline store selectors for `activeSessionId` and `removeSession`/`updateSessionTitle` (still needed for rename/delete/active highlighting). Keep `useStreamStore` for streaming indicator. Keep `useProjectContext` for knowing the current project.

    3. Remove the `useSessionList()` call -- `useMultiProjectSessions` handles fetching. Remove the `groupSessionsByDate` import -- grouping is now done inside the hook.

    4. Add scroll position preservation for collapse/expand:
    ```typescript
    const scrollRef = useRef<HTMLDivElement>(null);
    const handleProjectToggle = useCallback((projectName: string) => {
      const scrollTop = scrollRef.current?.scrollTop ?? 0;
      toggleProject(projectName);
      requestAnimationFrame(() => {
        if (scrollRef.current) scrollRef.current.scrollTop = scrollTop;
      });
    }, [toggleProject]);
    ```
    Attach `ref={scrollRef}` to the scrollable container div.

    5. Render structure (inside the scrollable div):
    ```
    {projectGroups.map((project) => (
      <div key={project.projectName}>
        <ProjectHeader
          displayName={project.displayName}
          sessionCount={project.visibleCount}
          isExpanded={expandedProjects.has(project.projectName)}
          onToggle={() => handleProjectToggle(project.projectName)}
          isCurrentProject={project.projectName === projectName}
        />
        {expandedProjects.has(project.projectName) && (
          <div>
            {project.dateGroups.map((dateGroup) => (
              <div key={dateGroup.label}>
                <DateGroupHeader label={dateGroup.label} />
                {dateGroup.sessions.map((session) => (
                  <SessionItem ... />  // Same props as current
                ))}
              </div>
            ))}
          </div>
        )}
      </div>
    ))}
    ```

    NOTE on collapse rendering: Use conditional rendering (`{isExpanded && ...}`) rather than CSS display:none. The research suggests CSS display toggle, but conditional rendering is simpler and the scroll preservation via rAF handles the position issue. The key anti-pattern to avoid is unmounting the SCROLL CONTAINER -- the scroll container stays mounted, only the inner content conditionally renders.

    6. The empty state should check if ALL projectGroups have zero visible sessions (not just if projectGroups is empty, since projects with only junk sessions should still show empty state).

    7. Delete and rename still operate on the timeline store for the CURRENT project. For sessions from OTHER projects, disable rename/delete in context menu (or don't show context menu). This keeps the existing flow intact -- when a cross-project session is clicked, the page navigates and reloads with the new project context.
