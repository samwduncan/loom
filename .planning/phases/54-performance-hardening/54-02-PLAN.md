  <action>
    Replace the current `useMemo([])` eager-mount pattern with a visit-tracking approach:

    1. Add a `useRef<Set<TabId>>` called `visitedTabs` initialized with `new Set(['chat', 'files'])` -- chat and files are always mounted eagerly (chat is the primary view, files is lightweight).

    2. In the render path, when `activeTab` changes, add it to `visitedTabs.current` (ref mutation in render is fine -- it's a monotonically-growing set, not state that drives re-renders).

    3. Replace the `useMemo` panels array with direct conditional rendering:
       ```
       - Chat: always rendered (primary panel)
       - Files: always rendered (lightweight, preserves tree state)
       - Shell: only rendered if visitedTabs.current.has('shell')
       - Git: only rendered if visitedTabs.current.has('git')
       ```

    4. For unvisited panels, render nothing (not even the skeleton -- the skeleton shows via Suspense when the panel IS visited but still loading).

    5. Keep the CSS show/hide pattern: each rendered panel div gets `className={activeTab === id ? 'h-full outline-none' : 'hidden'}`. The `hidden` class (display:none) preserves DOM but hides content.

    6. Keep the existing Suspense wrappers with TerminalSkeleton and GitPanelSkeleton fallbacks for when lazy chunks are loading.

    7. IMPORTANT: The `useMemo([])` is what causes eager mounting. By removing it and using conditional rendering based on visitedTabs, React.lazy() will only trigger the dynamic import when the JSX first appears in the render tree.
