**Stream store** (`stream.ts`): Add a `notifiedSessions: Set<string>` field to track sessions that completed in the background. Add actions:
- `addNotifiedSession(sessionId: string)` — adds to the set
- `clearNotifiedSession(sessionId: string)` — removes from the set (called when user views the session)

Add `notifiedSessions` to `INITIAL_STREAM_STATE` as `new Set<string>()`. In `reset()`, also reset `notifiedSessions` to a new empty Set.

**websocket-init.ts**: In the `onStreamEnd` callback, detect background completions. After the existing `streamStore().endStream()` call, check if the completed session is NOT the currently active/viewed session. Use `window.location.pathname` to get the currently viewed session ID (extract from `/chat/{id}` path). If the completed sessionId differs from the viewed session, call `streamStore().addNotifiedSession(sessionId)`.

Also handle live session data: in the `onLiveSessionData` callback, after adding messages, check if the sessionId is not the currently viewed session. If so, add to notifiedSessions.

Important: the `onStreamEnd` callback receives `_sessionId` and `_exitCode` parameters. The sessionId is currently unused (prefixed with `_`). Rename to `sessionId` and use it for the notification check.

**SessionItem** (`SessionItem.tsx`): Add `hasNewActivity?: boolean` to `SessionItemProps`. Render a notification dot when `hasNewActivity` is true and the session is not streaming/live/editing. Use CSS class `session-notified-dot`. Place it after the pin indicator and before the live/streaming dots in the dot priority chain:

```tsx
{hasNewActivity && !isStreaming && !isLiveAttached && !isEditing && (
  <span className="session-notified-dot" aria-label="New activity" />
)}
```

**SessionList** (`SessionList.tsx`): Subscribe to `notifiedSessions` from stream store:
```typescript
const notifiedSessions = useStreamStore((s) => s.notifiedSessions);
```

Pass `hasNewActivity={notifiedSessions.has(session.id)}` to each `<SessionItem>`.

Also: when a session is clicked (in `handleSessionClick`), clear its notification:
```typescript
const clearNotifiedSession = useStreamStore((s) => s.clearNotifiedSession);
// In handleSessionClick:
```
