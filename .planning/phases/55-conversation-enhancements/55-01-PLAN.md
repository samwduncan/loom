**FollowUpPills component** (`FollowUpPills.tsx`):

Create a component that generates contextual follow-up suggestions based on the last assistant message content. Props: `lastMessage: Message | null`, `onSelect: (text: string) => void`, `isStreaming: boolean`.

Client-side heuristic for suggestions (no backend endpoint per deferred decision). Parse the last assistant message content and derive 2-3 follow-up prompts:
- If message contains code blocks: suggest "Explain this code", "Refactor this", "Write tests for this"
- If message contains error-related words (error, exception, bug, fix, issue): suggest "What caused this?", "How do I prevent this?", "Show me the fix"
- If message contains list/steps: suggest "Go deeper on step 1", "Are there alternatives?", "Summarize this"
- Default fallback: "Tell me more", "Give me an example", "What should I do next?"

Pick 2-3 from the matching category. Do NOT show pills when `isStreaming` is true or when `lastMessage` is null or is a user message.

Render as a horizontal row of pill buttons styled consistently with the existing ChatEmptyState suggestion chips: `rounded-lg bg-surface-raised px-3 py-1.5 text-sm text-secondary hover:bg-surface-overlay transition-colors border border-border/10`. Use `cn()` for class composition.

Add `data-testid="follow-up-pills"` on the container and `data-testid="follow-up-pill"` on each button.

**follow-up-pills.css**: Minimal CSS for the pill row layout. Horizontal scrolling on mobile with `-webkit-overflow-scrolling: touch`, `scrollbar-width: none`, `gap: 8px`. On desktop, flex-wrap with centered alignment.
