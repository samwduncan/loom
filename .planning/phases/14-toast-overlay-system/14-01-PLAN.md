  <name>Task 2: Create overlay portal and mount Sonner provider</name>
  <files>index.html, src/components/ui/overlay-portal.tsx, src/App.tsx</files>
  <action>
1. In `index.html`, add an `#overlay-root` div after `#root` and before the script tag:

```html

```

2. Create `src/components/ui/overlay-portal.tsx`:

```tsx
import ReactDOM from 'react-dom';
import type { ReactNode } from 'react';

type OverlayPortalProps = {
  children: ReactNode;
};

/**
 * Portal wrapper that renders children into #overlay-root.
 * Ensures overlays escape all stacking contexts in the React tree.
 * Falls back to document.body if #overlay-root is not found.
 */
export function OverlayPortal({ children }: OverlayPortalProps) {
  const target = document.getElementById('overlay-root') || document.body;
  return ReactDOM.createPortal(children, target);
}
```

3. In `src/App.tsx`, add the Sonner Toaster component. Import `Toaster` from `'sonner'` and place it INSIDE the `<AuthProvider>` but as the LAST child before the closing `</AuthProvider>` tag (after the `</WebSocketProvider>` closing tag). This ensures toasts render above all content:

```tsx
  duration={3000}
  theme="dark"
  visibleToasts={4}
  offset={16}
/>
```

Note: Sonner creates its own portal to document.body automatically -- no need to wrap it in OverlayPortal. The CSS rules from Task 1 handle all theming. Keep the Toaster props minimal; styling is CSS-driven.
