1. Create `src/src/components/editor/EditorTabs.tsx`:
   - Read `openTabs`, `activeFilePath` from file store via selector hooks.
   - Read `setActiveFile`, `closeFile` from file store via selector hooks (separate selectors per Constitution 4.2).
   - Render a horizontal scrollable tab bar: `flex items-center gap-0 border-b border-border/8 bg-[var(--surface-base)] overflow-x-auto`.
   - Each tab renders:
     - Filename extracted via `path.split('/').pop()` (or `filePath.slice(filePath.lastIndexOf('/') + 1)`).
     - Full path in `title` attribute (tooltip on hover) (ED-09).
     - Dirty indicator: a small filled circle (4px, `var(--accent-primary)`) before the filename when `isDirty` is true (ED-05).
     - Close button (X icon, 12px) on the right side of each tab (ED-10).
     - Active tab has `bg-[var(--surface-raised)]` and `text-foreground`; inactive tabs have `text-muted-foreground hover:text-foreground`.
     - Tab styling: `px-3 py-1.5 text-xs font-[family-name:var(--font-mono)] flex items-center gap-1.5 cursor-pointer shrink-0 border-r border-border/8`.
   - Click tab → `setActiveFile(tab.filePath)`.
   - Click close button (with `e.stopPropagation()` to prevent tab switch):
     - If tab is NOT dirty → `closeFile(tab.filePath)` immediately.
     - If tab IS dirty → show AlertDialog with: "Save / Discard / Cancel" (ED-11).
       - Use the sibling AlertDialog pattern from Phase 21 (AlertDialog as sibling to avoid Radix focus trap issues).
       - "Save" → call save function (passed as prop or via hook), then closeFile on success.
       - "Discard" → closeFile directly (changes lost).
       - "Cancel" → close dialog, do nothing.
   - The close confirmation needs the save function. Accept `onSave: (filePath: string) => Promise<boolean>` as a prop from the parent (CodeEditor provides it from useFileSave).
   - When no tabs open, render nothing (the empty state is in CodeEditor).

   - Test: close button on clean tab calls closeFile immediately.
   - Test: dirty dot renders when isDirty is true.
   - Test: hovering tab shows full path via title attribute.
   - Test: empty openTabs renders nothing.
   - Mock file store with `vi.mock('@/stores/file')`. Use `@testing-library/react` render + screen + fireEvent.
