1. Create `src/src/hooks/useFileContent.ts`:
   - Hook: `useFileContent(projectName: string, filePath: string | null)`.
   - Returns `{ content: string | null, loading: boolean, error: string | null, isBinary: boolean, isLarge: boolean, proceed: () => void }`.
   - Binary detection: check file extension against `BINARY_EXTENSIONS` set (png, jpg, gif, mp3, mp4, pdf, zip, exe, dll, wasm, ttf, woff, woff2, etc). Note: images are already handled by `isImageFile()` in file-utils.ts and open in lightbox -- this covers non-image binaries.
   - Large file detection: accept optional `fileSize` param. If > 1MB (1_048_576 bytes), set `isLarge=true` and don't fetch until `proceed()` is called (`proceed` sets a `confirmed` state flag).
   - Fetch via `apiFetch<{ content: string; path: string }>(`/api/projects/${projectName}/file?filePath=${encodeURIComponent(filePath)}`)`.
   - Use `useEffect` with AbortController for cleanup. Set content/loading/error states.
   - On error, set `error` to message string (e.g. "File not found", "Permission denied").

2. Create `src/src/hooks/useFileSave.ts`:
   - Hook: `useFileSave(projectName: string)`.
   - Returns `{ save: (filePath: string, content: string) => Promise<boolean>, saving: boolean }`.
   - PUT via `apiFetch<{ success: boolean; path: string; message: string }>(`/api/projects/${projectName}/file`, { method: 'PUT', body: JSON.stringify({ filePath, content }) })`.
   - On success: show `toast.success('File saved')` via sonner, call `setDirty(filePath, false)` from file store, return true.
   - On error: show `toast.error('Save failed: ' + errorMessage)` with the backend error reason, return false.
   - Import `setDirty` via `useFileStore` selector hook (Constitution 4.2).
