    Add CSS keyframe animations to command-palette.css for the overlay and dialog mount.

    **Add these keyframes and animation rules:**

    ```css
    /* ─── Enter Transition (INTER-04) ─────────────────────────────
     * Fade + scale on mount. Exit is instant (command palette should feel snappy on dismiss).
     * Reduced motion handled by global override in base.css.
     */
    @keyframes cmdk-overlay-in {
      from { opacity: 0; }
      to { opacity: 1; }
    }

    @keyframes cmdk-content-in {
      from {
        opacity: 0;
        transform: scale(0.95) translateY(-10px);
      }
      to {
        opacity: 1;
        transform: scale(1) translateY(0);
      }
    }
    ```

    Then add `animation` property to the existing `[cmdk-overlay]` rule:
    ```css
    animation: cmdk-overlay-in var(--duration-normal) var(--ease-out);
    ```

    ```css
    animation: cmdk-content-in var(--duration-normal) var(--ease-out);
    ```

    **Do NOT add exit animation.** cmdk unmounts the dialog on close -- CSS exit animations would require a state manager wrapper, adding complexity for marginal benefit. Command palettes should feel instant on dismiss.

    **Do NOT add per-component @media (prefers-reduced-motion).** The global rule in base.css already sets `animation-duration: 0.01ms !important` for reduced motion.
