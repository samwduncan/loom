    **CRITICAL (SS-2): hideSplashWhenReady() must be async and await nativePluginsReady.**
    On cold start, `initializeNativePlugins()` is fire-and-forget. `splashModule` is still null when `hideSplashWhenReady()` runs synchronously after `createRoot().render()`. The function MUST await plugin readiness first.

    **CRITICAL (SS-1): Use one-argument subscribe form.**
    `useConnectionStore` uses only `persist` middleware, NOT `subscribeWithSelector`. The two-argument form `subscribe(selector, listener)` silently fails -- the selector becomes the listener, the actual listener is ignored. Use the one-argument form instead.

    Add new exported function `hideSplashWhenReady()`:
    ```typescript
    export async function hideSplashWhenReady(): Promise<void> {
      if (!IS_NATIVE) return;

      // SS-2: Wait for plugins to finish loading before checking splashModule.
      // initializeNativePlugins() is fire-and-forget from main.tsx, so splashModule
      // may still be null when this function is called.
      await nativePluginsReady;

      if (!splashModule) return;

      let dismissed = false;
      const dismiss = () => {
        if (dismissed) return;
        dismissed = true;
        splashModule?.SplashScreen.hide({ fadeOutDuration: 300 });
        unsubscribe();
        if (splashFallbackTimer !== null) {
          clearTimeout(splashFallbackTimer);
          splashFallbackTimer = null;
        }
      };

      // SS-1: One-argument subscribe form. The store uses `persist` middleware only,
      // NOT `subscribeWithSelector`. The two-argument form silently fails.
      const unsubscribe = useConnectionStore.subscribe((state) => {
        if (state.providers.claude.status === 'connected') dismiss();
      });

      // Check current state immediately (connection may already be established)
      const currentState = useConnectionStore.getState();
      if (currentState.providers.claude.status === 'connected') {
        dismiss();
        return;
      }

      // D-10: Fallback timeout -- hide after 3s even if connection fails
      splashFallbackTimer = setTimeout(dismiss, 3000);
    }
    ```

    Import `useConnectionStore` from `@/stores/connection` at the top of the file.

    **IMPORTANT (S-4): Do NOT add `touch-action: pan-y` to `.app-shell` or anywhere else.**
    `touch-action: pan-y` suppresses horizontal touch events, which breaks:
    1. The sidebar's swipe-to-close gesture handler (tracks deltaX)

    Instead, rely on the existing combination: `html { overflow: hidden }` + `body { overflow: hidden }` + AppShell's `h-dvh overflow-hidden` + `overscroll-behavior: none` on html/body. This is sufficient for iOS 16+ rubber-band prevention.
