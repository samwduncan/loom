  <action>
    Audit every interactive element and add missing ARIA attributes. Specific fixes required:

    **Sidebar:**
    - SessionItem: add `role="option"` and `aria-selected={isActive}` to the session row div. Add `aria-label` with session title.
    - SessionList: add `role="listbox"` and `aria-label="Chat sessions"` to the scrollable session container.
    - NewChatButton: verify `aria-label="New chat"` exists (likely needs adding).
    - QuickSettingsPanel: ensure the popover trigger button has `aria-expanded`, `aria-haspopup="dialog"`, and `aria-label="Quick settings"`.
    - ConnectionStatusIndicator: add `aria-label` describing current status (e.g., "Connection status: connected").

    **File Tree:**
    - FileNode: add `role="treeitem"`, `aria-expanded` (for directories), `aria-selected={isActive}`. The tree container (FileTreePanel) should have `role="tree"` and `aria-label="Project files"`.
    - FileTreePanel: add `role="tree"` to the tree container div, `aria-label="Project files"`.

    **Chat Components:**
    - ToolChip: the clickable chip div needs `role="button"`, `tabIndex={0}`, `aria-expanded={isExpanded}`, `aria-label` like "Tool: Read file.txt - completed".
    - ToolCallGroup: the expand/collapse header needs `aria-expanded` and `aria-label`.
    - ChatComposer: the textarea should have `aria-label="Message input"`. Send button needs `aria-label="Send message"`. Stop button needs `aria-label="Stop generation"`.
    - MentionPicker: add `role="listbox"`, `aria-label="File suggestions"`. Each item gets `role="option"`, `aria-selected`.
    - SlashCommandPicker: add `role="listbox"`, `aria-label="Slash commands"`. Each item gets `role="option"`, `aria-selected`.
    - MentionChipRow: each chip's remove button needs `aria-label="Remove mention: {filename}"`.
    - ImagePreviewCard: the remove button needs `aria-label="Remove image: {filename}"`.
    - CodeBlock: the copy button needs `aria-label="Copy code"`. The language label span should be presentational.
    - ThinkingDisclosure: the toggle button needs `aria-expanded` and `aria-label="Toggle thinking content"`.
    - ScrollToBottomPill: add `aria-label="Scroll to bottom"` with unread count if present (e.g., "Scroll to bottom, 5 new messages").
    - CollapsibleMessage: collapsed state button needs `aria-label="Expand collapsed message"`.
    - TokenUsage: the expand/collapse toggle needs `aria-expanded` and `aria-label`.

    **Git Panel:**
    - ChangedFileRow: the stage/unstage button needs `aria-label="Stage file: {path}"` or `"Unstage file: {path}"`. The file path should have `aria-label` with full path.
    - CommitRow: add `role="listitem"` if in a list. Expand button needs `aria-expanded` and `aria-label`.
    - CommitComposer: the commit message textarea needs `aria-label="Commit message"`. Commit button needs `aria-label="Create commit"`.
    - BranchSelector: ensure the select/dropdown has `aria-label="Branch selector"`.

    **Terminal:**
    - TerminalHeader: the connection status dot should have `aria-label` describing terminal connection status.

    **Editor:**
    - EditorTabs: each tab should have `role="tab"`, `aria-selected`, and `aria-label` with filename. Close buttons need `aria-label="Close {filename}"`.

    Do NOT change component behavior -- only add ARIA attributes. Keep all existing data-testid attributes.

    Implement WAI-ARIA Tabs pattern on TabBar:
