Build the TurnBlock grouping system that transforms flat message lists into collapsible, navigable AI turns.

Purpose: CHAT-03 is the most architecturally significant change in Phase 5 -- introducing the concept of "turns" that group consecutive AI messages (assistant + tool use + thinking) into collapsible blocks. This makes agentic sessions with 20+ turns navigable without scrolling through every tool call. The turn grouping hook derives structure from the existing flat ChatMessage[] without mutating data.

Output: useTurnGrouping hook, TurnBlock component, TurnToolbar, updated ChatMessagesPane rendering.

     id: string;
     messages: ChatMessage[];
     startTime: Date;
     endTime?: Date;
     isStreaming: boolean;

     failedToolCount: number;

2. Create `src/components/chat/hooks/useTurnGrouping.ts`:
   - Export `useTurnGrouping(messages: ChatMessage[]): { items: (ChatMessage | Turn)[], turnCount: number }`
   - Uses `useMemo` to derive turns from flat messages -- recomputes only when messages array reference changes
   - Grouping algorithm:
     a. Iterate through messages
     b. User messages (`type === 'user'`) are standalone -- flush any accumulated assistant messages as a Turn, push user message
     c. Assistant messages, tool messages, thinking messages, error messages accumulate into the current turn
     d. A turn ends when a user message appears or messages end
   - Turn ID: use timestamp of first message in the turn as a stable ID (e.g., `turn-${startTime.getTime()}`)
   - `firstProseContent`: scan turn messages for first `type === 'assistant'` message with non-empty content that is NOT `isToolUse` and NOT `isThinking`. Extract first line (up to 80 chars, ellipsized).
   - `toolCallCount`: count messages where `isToolUse === true`
   - `failedToolCount`: count messages where `isToolUse && toolResult?.isError`
   - `isStreaming`: true if any message in the turn has `isStreaming === true`
   - `durationMs`: difference between last message timestamp and first message timestamp (only when turn is not streaming)

3. Create `src/components/chat/view/subcomponents/TurnToolbar.tsx`:
   - Props: `{ turnCount: number, onExpandAll: () => void, onCollapseAll: () => void }`
   - Only renders when `turnCount >= 2` (no toolbar for single-turn conversations)
   - Compact bar at top of chat area: "N turns" label + "Expand all" / "Collapse all" buttons
   - Styling: small text, muted, positioned sticky below the message loading indicators
   - Use lucide-react `ChevronsDown` (expand) and `ChevronsUp` (collapse) icons
