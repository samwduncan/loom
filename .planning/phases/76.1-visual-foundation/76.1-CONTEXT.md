**Gathered:** 2026-04-04

<domain>
## Phase Boundary

Clone ChatGPT/Claude iOS visual language at pixel level across three surfaces: drawer, chat screen, and composer. Fix surface color hierarchy for device readability, add glass surfaces via expo-blur, match reference app density and spacing, and ensure all interactions have spring animation + haptic feedback. <decisions>
## Implementation Decisions

### Surface Color Strategy
- **D-01:** Keep the device-optimized wider RGB jumps currently in `mobile/lib/colors.ts`. The Soul doc's original 8-unit jumps were tested on real hardware and found to be imperceptible at arm's length. Current values (16-20 unit jumps) provide clearly visible surface hierarchy. Update NATIVE-APP-SOUL.md to match the working code values, not the other way around.
- **D-02:** Minimum 12-15 RGB unit gap between adjacent surface tiers. Current values already exceed this. The quality test is: at 50% screen brightness on iPhone, can you distinguish raised from base? If not, widen the gap.
- **D-03:** Add `rgba(255,255,255,0.05)` top-edge "rim light" border on raised surfaces. This 0.5pt border provides more perceived separation than RGB jumps alone — especially on screens with fingerprint smudges. Bard's recommendation, validated against reference apps where ChatGPT uses subtle edge highlights.

### Glass Surfaces (VIS-02)
- **D-04:** Glass via `expo-blur` BlurView on navigation header and floating composer. Intensity 40, background tint `dark`, overlay `rgba(0,0,0,0.35)`, border 1px `border-subtle`, corner radius 16px. Exactly per Soul doc spec.
- **D-05:** Drawer search input: glass treatment as stretch goal, not mandatory. Drawer background is `surface-sunken` and content rarely scrolls under the search — glass overhead isn't justified there unless it looks noticeably better. Claude's discretion based on visual testing.
- **D-06:** Performance budget: maintain 55+ FPS with blur active during message streaming. If FPS drops on older devices (iPhone 12 or earlier), fall back to opaque `surface-raised` with `border-subtle` border. The fallback must look intentional, not broken.
- **D-07:** Keyboard-blur collision risk: the composer BlurView moving with the keyboard is a known React Native pain point (blur can lag 1-2 frames behind keyboard animation). Prototype this interaction early — if it stutters, use opaque composer with a top border instead. Performance > glass.

### Drawer Density (VIS-03)
- **D-08:** Clone ChatGPT iOS drawer information density. Key metrics to match: session item height (~56px), padding (px-4 py-3), title font (15px semibold), subtitle font (12px regular), section header capitalization/spacing. Current DrawerContent already uses these values — verify on device, adjust where off.
- **D-09:** Drawer width: 80% of screen width, capped at 320px. Current implementation should already match — verify.
- **D-10:** "New Chat" button prominence: accent background, 44px height, full drawer width minus margins. Matches ChatGPT's prominent new chat button at top of drawer.

### Chat Screen Feel (VIS-04)
- **D-11:** Clone ChatGPT/Claude iOS message density. Key metrics: user bubble padding (p-4, rounded-2xl), assistant message padding (none — flush on base), turn spacing (24px between roles, 8px between same-role), avatar size (24px). Current code matches these specs from Phase 75 context — verify on device.
- **D-12:** Navigation header: centered title (17px semibold), hamburger left, model indicator right. Add glass background per D-04.
- **D-13:** Streaming indicator: 2px accent line pulsing at bottom of message area. Already spec'd in Phase 75 — verify implementation and visual quality.

### Interaction Weight & Haptic Hierarchy
- **D-14:** NOT blanket haptics on every tap. Use a **haptic hierarchy** mapped to interaction importance:
  - **Impact Light**: button press feedback, selection, scroll-to-bottom tap
  - **Impact Medium**: drawer open/close snap, state transitions (composer idle→typing), session switch
  - **Success Notification**: tool call complete, session complete
  - **Warning Notification**: permission request appears
  - **Error Notification**: error state transition
  - **Selection (light)**: swipe threshold crossed, list item long-press
- **D-15:** All interactive elements respond to press on **touch-start** (not touch-end) with Micro spring scale (0.97). This is already implemented in most Phase 75 components — audit and add where missing.
- **D-16:** Composer FSM state transitions (idle→typing, typing→sending, active→idle) should each have distinct haptic + visual feedback. The "typing begins" moment is where the app feels alive.

### Spring Animation Audit
- **D-17:** Audit all existing components for missing spring animations. Every appearance, disappearance, and state change must use named springs from `springs.ts`. No linear/ease animations except keyboard avoidance (which must match iOS system curve).
- **D-18:** Drawer "thud" haptic synchronized with spring rest state — when the drawer hits fully open or fully closed, fire Impact Medium haptic at that exact frame.

### Claude's Discretion
- Exact opacity values for glass overlay (start with spec, tune on device)
- Whether dynamic color shifts during streaming are implemented in this phase or left for Phase 84+ (they're spec'd but might be v6.0 "The Polish" scope)
- Order of implementation across surfaces (drawer first vs chat first vs composer first)
- Whether to add scroll-position-dependent glass tint intensity on chat header (Bard's suggestion — deeper tint as user scrolls, creating depth perception)
- rim light border width (0.5pt vs 1pt — test on device)

</decisions>

<specifics>
## Specific Ideas

- **"Clone the Foundation" (Phase 1 of 3)**: This phase IS the foundation cloning from the design philosophy doc. We are at Phase 1 (clone reference apps) — not Phase 2 (integrate unique features) or Phase 3 (elevate beyond). The goal is to reach parity, not exceed it.
- **Vertical rhythm is everything**: ChatGPT iOS succeeds because of precise vertical rhythm — the whitespace between bubble and screen edge, the tap target sizes, the scroll physics. This matters more than exact hex codes.
- **Side-by-side test is the exit criteria**: "User opens app next to ChatGPT iOS — drawer, chat, and composer look like they belong in the same tier." This is the literal success criterion. Everything is judged against this test.
- **Performance is a visual feature**: Zero-stutter scrolling with a blurred header matters more than perfect RGB compliance. If glass causes jank, glass goes. 55+ FPS is non-negotiable.
- **swd's historical feedback**: "Everything is too big", "Flat, static, and bland", "Really doesn't feel like it's built well for iOS." This phase directly addresses these complaints.

</specifics>

<quality_bar>
## Quality Bar (Bard Assessment)

**Good:**
- Surface colors create visible hierarchy (you can see the difference between tiers)
- Glass surfaces blur behind header and composer
- Springs on most button presses
- Haptics on send/delete/approval

**Exceptional (what separates world-class):**
- **Haptic hierarchy**: Light selection taps for tool chips, medium impact for message sent, success notification when tool finishes, warning notification for permission request. Not blanket buzzing — musical haptics.
- **Spring synchronicity**: Drawer doesn't just slide — it has slight overshoot and a haptic "thud" exactly when the spring reaches rest state. The physics and haptics are frame-synchronized.
- **Zero-latency response**: Button press feedback on touch-START, not touch-end. The button visually "depresses" immediately. No perceived input lag.
- **Rim light separation**: Raised surfaces have a subtle 0.5pt top-edge highlight (`rgba(255,255,255,0.05)`) that provides depth perception even on smudged screens — more effective than raw RGB difference.
- **Performance under load**: 60 FPS maintained while streaming tokens through a blurred header. The glass effect doesn't cost anything the user can see.
- **Vertical rhythm**: Session items, message bubbles, and composer all match ChatGPT iOS density so closely that a side-by-side screenshot looks like the same design language.

**The quality bar is:** Put the app next to ChatGPT iOS at arm's length. A non-technical person cannot tell which one is the "indie" app. The density, the motion, the surface depth — they belong in the same league.

</quality_bar>
