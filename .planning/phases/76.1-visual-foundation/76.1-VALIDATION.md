## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Surface hierarchy visible at arm's length | VIS-01 | Visual perception test on real hardware | At 50% screen brightness on iPhone, verify all 4 surface tiers are distinguishable. Compare against NATIVE-APP-SOUL.md spec. |
| Glass blur on header and composer | VIS-02 | GPU compositing + FPS measurement | Open chat with messages, verify blur behind header. Stream tokens, verify 55+ FPS maintained. Test keyboard-blur on composer. |
| Drawer density matches ChatGPT iOS | VIS-03 | Side-by-side visual comparison | Screenshot Loom drawer next to ChatGPT iOS drawer. Compare: item height (~56px), padding, font sizes, section headers. |
| Chat screen matches reference feel | VIS-04 | Side-by-side visual comparison | Screenshot Loom chat next to ChatGPT iOS chat. Compare: bubble padding, turn spacing, avatar size, header layout. |
| Spring animations on all interactions | VIS-04 | Animation timing requires human evaluation | Tap buttons, open drawer, switch sessions — all must use spring physics. No linear/ease except keyboard avoidance. |
| Haptic hierarchy correct | VIS-04 | Haptic feedback requires physical device | Verify: Light on selection, Medium on drawer snap, Success on tool complete, Warning on permission request. |

**Justification:** All four VIS requirements are visual quality assessments requiring human judgment against reference app screenshots. The success criterion is explicitly "User opens app next to ChatGPT iOS." Automated pixel-diff testing is Phase 84 scope (Maestro + Kimi).
