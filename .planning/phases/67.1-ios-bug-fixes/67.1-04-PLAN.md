**Keyboard Animation Sync (Forgejo #3):**

The current `useKeyboardOffset.ts` is a no-op on native (lines 30-33: `if (IS_NATIVE) return;`). WKWebView's default Body resize handles keyboard avoidance, but the resize happens AFTER the keyboard animation completes -- not in sync. This causes a visible gap between keyboard appearing and composer moving up.

Rewrite the native path in `useKeyboardOffset.ts` to use Capacitor Keyboard events (`keyboardWillShow`/`keyboardWillHide`) which fire at the START of the keyboard animation:

```typescript
export function useKeyboardOffset(): void {
  useEffect(() => {
    if (IS_NATIVE) return setupNativeKeyboardOffset();
    return setupWebFallback();
  }, []);
}

/**
 * Native path: Capacitor Keyboard events for animation-synced offset.
 *
 * Uses keyboardWillShow/keyboardWillHide events which fire at the START of
 * the keyboard animation (not after completion like body resize).
 *
 * NOTE: This works WITH WKWebView's default Body resize (KeyboardResize.Body),
 * not instead of it. The Body resize handles viewport shrinking. The
 * --keyboard-offset variable enables CSS transitions that sync with the
 * keyboard animation timing.
 *
 * We do NOT change KeyboardResize mode to None (pitfall: double-resize).
 * Instead, --keyboard-offset is used as a supplementary signal for
 * CSS transition timing. The composer uses a CSS transition on transform
 * that matches the keyboard animation duration (~250ms).
 */
async function setupNativeKeyboardOffset(): Promise<(() => void) | undefined> {
  await nativePluginsReady;
  const mod = getKeyboardModule();
  if (!mod) return undefined;

  const { Keyboard } = mod;

  const willShowHandle = await Keyboard.addListener('keyboardWillShow', (info: { keyboardHeight: number }) => {
    document.documentElement.style.setProperty('--keyboard-offset', `${info.keyboardHeight}px`);
  });

  const willHideHandle = await Keyboard.addListener('keyboardWillHide', () => {
    document.documentElement.style.setProperty('--keyboard-offset', '0px');
  });

  return () => {
    void willShowHandle.remove();
    void willHideHandle.remove();
  };
}
```

Add the necessary imports at the top of `useKeyboardOffset.ts`:
```typescript
import { IS_NATIVE } from '@/lib/platform';
import { nativePluginsReady, getKeyboardModule } from '@/lib/native-plugins';
```

**IMPORTANT WARNING from research:** The `keyboardWillShow` approach works WITH the default Body resize, not instead of it. Do NOT change `KeyboardResize` mode in native-plugins.ts. The `--keyboard-offset` CSS variable provides timing information for CSS transitions. If device testing shows a "double animation" effect (content jumps down then bounces back), the fix is to remove the native path and revert to the no-op -- but try this approach first as the research recommends it.

**PTR Animation Improvement (Forgejo #13):**

The current PullToRefreshSpinner uses `pullDistance * 0.6` for height during pulling, which creates an abrupt linear transition. On release-to-refresh, the height snaps from the pull position to 40px with a basic ease, creating a jolt.

**1. Update PullToRefreshSpinner.tsx:**

Replace the linear height calculation with an elastic resistance curve that decelerates as the user pulls further (natural rubber-band feel):

```typescript
export function PullToRefreshSpinner({ pullDistance, isRefreshing, threshold }: PullToRefreshSpinnerProps) {
  if (pullDistance <= 0 && !isRefreshing) return null;

  const progress = Math.min(pullDistance / threshold, 1);
  const opacity = isRefreshing ? 1 : Math.min(progress * 1.5, 1); // Fade in faster
  const rotation = isRefreshing ? undefined : progress * 270;

  // Elastic resistance: decelerates as user pulls further (rubber-band feel).
  // At threshold (60px pull), height is ~36px. Diminishing returns beyond threshold.
  const elasticHeight = isRefreshing
    ? 40
    : Math.min(60, pullDistance * 0.6 * (1 - Math.pow(Math.max(0, 1 - pullDistance / 120), 2) * 0.3));

  return (
    <div
      className="flex items-center justify-center overflow-hidden"
      // eslint-disable-next-line loom/no-banned-inline-style -- gesture-driven dynamic values
      style={{
        height: elasticHeight,
        opacity,
        // Spring transition on release-to-refresh state change.
        // No transition during active pull (gesture-driven, must be immediate).
        transition: isRefreshing
          ? 'height var(--duration-spring-snappy) var(--ease-spring-snappy), opacity 150ms ease-out'
          : 'none',
      }}
    >
      <svg
        width="24"
        height="24"
        viewBox="0 0 24 24"
        fill="none"
        className={cn(isRefreshing && 'animate-spin')}
        style={{
          animationDuration: '800ms',
          ...(!isRefreshing && rotation !== undefined
            ? { transform: `rotate(${rotation}deg)` }
            : {}),
        }}
      >
        <circle
          cx="12" cy="12" r="10"
          stroke="var(--accent-primary)"
          strokeWidth="2.5"
          strokeLinecap="round"
          strokeDasharray={isRefreshing ? '45 18' : `${progress * 63} ${63 - progress * 63}`}
          fill="none"
        />
      </svg>
    </div>
  );
}
```

Key changes:
- `elasticHeight` uses an elastic resistance curve instead of linear `pullDistance * 0.6`
- `overflow-hidden` on the container prevents spinner from overflowing during transitions
- Release-to-refresh transition uses `var(--duration-spring-snappy) var(--ease-spring-snappy)` for bounce feel
- Opacity fades in faster (`progress * 1.5`) so spinner is visible sooner
- `strokeWidth` increased from 2 to 2.5 for better mobile visibility
