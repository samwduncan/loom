## S+ Issues (Must Fix)

### [SS] Plan 03: `e.preventDefault()` in useLongPress touchstart breaks scrolling
**Source:** All 4 agents | **Confidence:** High
**Plan:** 67.1-03, Task 1
**Description:** Calling `e.preventDefault()` in touchstart suppresses ALL default touch behavior including vertical scrolling. Since useLongPress wraps every session item and message bubble, the session list and message list become unscrollable on iOS.
**Suggested fix:** Remove `e.preventDefault()` from handleTouchStart entirely. Use CSS `-webkit-user-select: none; -webkit-touch-callout: none` via `[data-long-press-target]` attribute (already in the plan) to suppress text selection. For messages (no data-long-press-target), accept temporary selection during 500ms timer and cancel via touchmove threshold.

### [S] Plan 04: Keyboard offset blocked by `padding-bottom: 0 !important`
**Source:** Hunter | **Confidence:** High
**Plan:** 67.1-04, Task 1
**Description:** `base.css` line 240 has `html[data-native] .app-shell { padding-bottom: 0 !important; }` which overrides `var(--keyboard-offset)`. The CSS variable will be set but have zero visual effect. Forgejo #3 remains unfixed.
**Suggested fix:** Add `base.css` to read_first and files_modified. The plan must either remove the `!important` override for native or use a different mechanism (CSS transform on composer).

### [S] Plan 04: Async cleanup from `setupNativeKeyboardOffset` leaks listeners
**Source:** Guard, Hunter, Bard | **Confidence:** High
**Plan:** 67.1-04, Task 1
**Description:** `setupNativeKeyboardOffset` is async and returns `Promise<(() => void) | undefined>`. React useEffect cleanup must be synchronous. The Promise is not callable, so Capacitor listeners are never removed on unmount.
**Suggested fix:** Use ref pattern: `let cleanup; setupNativeKeyboardOffset().then(fn => { cleanup = fn; }); return () => { cleanup?.(); };`
