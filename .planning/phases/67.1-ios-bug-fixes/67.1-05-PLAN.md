**Page Reload Investigation (Forgejo #5):**

The research identifies two possible causes:
1. **WKWebView process termination under memory pressure** -- iOS kills the web content process and Capacitor auto-reloads. This triggers `webViewWebContentProcessDidTerminate` in the native layer.
2. **Capacitor live reload / HMR** -- if the server.url mode has HMR enabled, the Vite dev server may trigger hot reloads.

**Step 1: Check Capacitor config for live reload signals.**

In `capacitor.config.ts`, verify that `server.url` is NOT set to a Vite dev server URL with HMR. Currently:
```typescript
url: process.env.CAPACITOR_SERVER_URL || undefined,
```
This is fine -- it's only set when explicitly configured. But the Vite dev server at port 5184 includes HMR by default. If the iOS app is pointed at the dev server, HMR WebSocket messages could trigger full reloads when files change during development.

Add a comment documenting this:
```typescript
server: {
  // When pointing to the Vite dev server (e.g., http://100.86.4.57:5184),
  // HMR WebSocket messages may trigger page reloads during active development.
  // This is expected behavior in development but should NOT occur in production
  // (bundled assets mode). If random reloads occur in production, investigate
  // WKWebView memory pressure (webViewWebContentProcessDidTerminate in Xcode console).
  url: process.env.CAPACITOR_SERVER_URL || undefined,
  cleartext: true,
  allowNavigation: ['100.86.4.57:*', '*.ts.net'],
},
```

**Step 2: Add reload detection logging.**

In `native-plugins.ts`, add a reload detection mechanism at the top of `initializeNativePlugins()`, BEFORE the init guard:

```typescript
export async function initializeNativePlugins(): Promise<void> {
  // Reload detection: log when native plugins initialize (helps diagnose #5 page reloads).
  // If this fires multiple times in Xcode console, the page is reloading.
  if (IS_NATIVE) {
    const now = new Date().toISOString();
    console.log(`[native-plugins] initializeNativePlugins called at ${now}. isInitialized=${isInitialized}`);
  }

  if (isInitialized || !IS_NATIVE) {
    resolveReady();
    return;
  }
  // ... rest unchanged ...
}
```

Also add a `beforeunload` listener to catch when the page is about to reload. IMPORTANT: Guard this behind `IS_NATIVE` so it only runs on native builds (not permanent prod code on web). Add cleanup support to `_resetForTesting`:

```typescript
// Add at module level, after the nativePluginsReady declaration:
let beforeUnloadHandler: (() => void) | null = null;

if (typeof window !== 'undefined' && IS_NATIVE) {
  beforeUnloadHandler = () => {
    console.warn('[native-plugins] Page is being unloaded/reloaded. If unexpected, check Xcode console for webViewWebContentProcessDidTerminate or HMR triggers.');
  };
  window.addEventListener('beforeunload', beforeUnloadHandler);
}
```

Update `_resetForTesting()` to clean up the beforeunload listener:
```typescript
export function _resetForTesting(): void {
  isInitialized = false;
  keyboardModule = null;
  splashModule = null;
  setHapticsModule(null);
  if (splashFallbackTimer !== null) {
    clearTimeout(splashFallbackTimer);
    splashFallbackTimer = null;
  }
  if (beforeUnloadHandler) {
    window.removeEventListener('beforeunload', beforeUnloadHandler);
    beforeUnloadHandler = null;
  }
  nativePluginsReady = new Promise<void>((resolve) => {
    resolveReady = resolve;
  });
}
```
