## Decisions Made
- **useEffect for ref sync**: React `react-hooks/refs` ESLint rule prevents assigning to ref.current during render. Used useEffect to sync onLongPressRef instead.
- **No e.preventDefault()**: Deliberately omitted from touchstart to preserve vertical scrolling on both lists. iOS text selection/callout suppressed via CSS data attribute instead.
- **data-long-press-target on sessions only**: Messages must preserve text selection, so the CSS suppression attribute is only applied to session items.
- **Separate CSS class names**: .popover-menu-item in base.css avoids cascade conflict with .context-menu-item already defined in sidebar.css (different layout properties).
- **Arrow-key navigation**: Manual div[role="menuitem"] elements in Popover don't get native arrow-key management like Radix ContextMenuItem, so handleMenuKeyDown was added for keyboard accessibility.
