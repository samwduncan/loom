  <files>src/src/lib/websocket-client.ts, src/src/lib/websocket-client.test.ts</files>
  <behavior>
    - WebSocketClient class manages a single WebSocket connection
    - connect(token) opens ws to relative /ws?token=<jwt> URL (resolved from window.location)
    - Connection state machine: disconnected -> connecting -> connected -> (reconnecting -> connecting -> connected)
    - On open: state = connected, reset reconnect attempts, send get-active-sessions, call onStateChange callback
    - On close: if was connected/reconnecting, schedule reconnect; update state
    - On error: set provider error in connection store
    - On message: JSON.parse, validate via isServerMessage, call onMessage callback
    - scheduleReconnect uses exponential backoff: min(1000 * 2^attempts, 30000)
    - disconnect() clears timers, closes socket, sets state to disconnected
    - send(msg) returns false if not connected, true if sent
    - subscribeContent(listener) returns unsubscribe function
    - Exported singleton: export const wsClient = new WebSocketClient()
    - Malformed messages are logged and skipped (console.warn)
    - Test: connect triggers state to connected
    - Test: close triggers reconnect schedule
    - Test: send returns false when disconnected
    - Test: message parsing dispatches to onMessage
    - Test: subscribeContent + unsubscribe works
  </behavior>
  <action>
    Create `src/src/lib/websocket-client.ts`:
    - Import types: ServerMessage, ClientMessage, isServerMessage from @/types/websocket
    - Define type `ConnectionState = 'disconnected' | 'connecting' | 'connected' | 'reconnecting'`
    - Class `WebSocketClient`:
      - Private fields: ws (WebSocket|null), state (ConnectionState), reconnectAttempts (number), reconnectTimer (ReturnType<typeof setTimeout>|null), token (string|null), maxReconnectDelay (30_000)
      - Private callback fields: onMessage ((msg:ServerMessage)=>void)|null, onStateChange ((state:ConnectionState)=>void)|null
      - Private contentListeners: Set<(token:string)=>void>
      - Private streamBuffer: string[] -- accumulates content tokens when no listeners are subscribed
      - Private isStreamActive: boolean -- tracks whether a stream is in progress (for buffer lifecycle)
      - `configure(callbacks: {onMessage, onStateChange})` -- set callback fields. Called once during app init.
      - `connect(token: string)`: Build WS URL from `window.location` (protocol ws/wss based on location.protocol, host from location.host, path `/ws?token=`). Set state to 'connecting', call onStateChange. Create WebSocket. Wire onopen/onclose/onerror/onmessage handlers.
      - `disconnect()`: Clear reconnect timer. Close socket with code 1000. Set state to 'disconnected'. Call onStateChange. Set ws to null.
      - `send(msg: ClientMessage): boolean`: If state !== 'connected' or !ws, return false. ws.send(JSON.stringify(msg)). Return true.
      - `getState(): ConnectionState`: Return current state.
      - `subscribeContent(listener: (token:string)=>void): ()=>void`: Add to contentListeners. If streamBuffer has accumulated tokens, replay them immediately to the new listener (flush backlog). Return function that deletes the listener.
      - `emitContent(token: string)`: If contentListeners.size > 0, calls all listeners with the token. If no listeners, pushes token to streamBuffer (backlog for late subscribers).
      - `startContentStream()`: Set isStreamActive = true, clear streamBuffer. Called by websocket-init when a new stream begins.
      - `endContentStream()`: Set isStreamActive = false. Keeps buffer contents (late subscriber can still replay). Buffer cleared on next startContentStream.
      - Private `handleOpen()`: Set state to 'connected'. Reset reconnectAttempts to 0. Call onStateChange. Send get-active-sessions via send().
      - Private `handleClose(event: CloseEvent)`: If state was 'connected' or 'reconnecting' and event.code !== 1000 (normal closure), schedule reconnect. Otherwise set state to 'disconnected' + onStateChange.
      - Private `handleError()`: Call onStateChange with current state (no transition -- the close event handles it).
      - Private `handleMessage(event: MessageEvent)`: Try JSON.parse. Validate with isServerMessage. If valid, call onMessage. If invalid, console.warn and return.
      - Private `scheduleReconnect()`: Set state to 'reconnecting'. Call onStateChange. Increment reconnectAttempts. Calculate delay = min(1000 * 2^reconnectAttempts, maxReconnectDelay). Set timeout to call reconnect().
      - Private `reconnect()`: If no token, return. Set state to 'connecting'. Call onStateChange. Create new WebSocket with stored token.
    - Export `const wsClient = new WebSocketClient()`
