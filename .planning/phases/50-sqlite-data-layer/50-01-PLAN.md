**Step 1: Create `server/cache/schema.sql`**

```sql
-- Schema version 1 (tracked via PRAGMA user_version)
-- This file is the DDL for cache.db, a DISPOSABLE cache of JSONL data.
-- Deleting cache.db causes rebuild from JSONL on next request.

CREATE TABLE IF NOT EXISTS sessions (
  id TEXT PRIMARY KEY,
  project_name TEXT NOT NULL,
  summary TEXT DEFAULT 'New Session',
  message_count INTEGER DEFAULT 0,
  last_activity TEXT,
  cwd TEXT,
  last_user_message TEXT,
  last_assistant_message TEXT,
  jsonl_file TEXT,
  jsonl_mtime REAL,
  jsonl_size INTEGER,
  cached_at TEXT NOT NULL,
  is_junk INTEGER DEFAULT 0
);

CREATE TABLE IF NOT EXISTS messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id TEXT NOT NULL,
  project_name TEXT NOT NULL,
  entry_type TEXT NOT NULL,
  message_id TEXT,
  uuid TEXT UNIQUE,
  parent_uuid TEXT,
  timestamp TEXT,
  raw_json TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_sessions_project ON sessions(project_name, last_activity DESC);
CREATE INDEX IF NOT EXISTS idx_sessions_not_junk ON sessions(project_name, is_junk, last_activity DESC);
CREATE INDEX IF NOT EXISTS idx_messages_session ON messages(session_id, timestamp ASC);
CREATE INDEX IF NOT EXISTS idx_messages_type ON messages(session_id, entry_type);
CREATE INDEX IF NOT EXISTS idx_messages_uuid ON messages(uuid);
```

**Step 2: Create `server/cache/message-cache.js`**

Create a class-based module that exports a singleton `MessageCache` instance:

```javascript
import Database from 'better-sqlite3';
import path from 'path';
import fs from 'fs';
import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

const SCHEMA_VERSION = 1;
```

**Class: MessageCache**

Constructor:
- Accepts optional `dbPath` (defaults to `path.join(__dirname, '..', 'database', 'cache.db')`)
- Ensures parent directory exists (`fs.mkdirSync({ recursive: true })`)
- Opens better-sqlite3 Database
- Sets pragmas: `journal_mode = WAL`, `synchronous = NORMAL`, `cache_size = -64000` (64MB), `temp_store = MEMORY`, `foreign_keys = ON`, `busy_timeout = 5000`
- Checks `PRAGMA user_version` — if less than `SCHEMA_VERSION`, drops all tables and recreates from schema.sql. Then sets `PRAGMA user_version = ${SCHEMA_VERSION}`.
- Prepares commonly-used statements and stores on `this._stmts` for reuse

**Methods:**

`upsertSession(session)` — Accepts object `{ id, projectName, summary, messageCount, lastActivity, cwd, lastUserMessage, lastAssistantMessage, jsonlFile, jsonlMtime, jsonlSize }`. Uses `INSERT OR REPLACE INTO sessions ...` with `cached_at = new Date().toISOString()`. Returns void.

`upsertSessionBatch(sessions)` — Wraps multiple `upsertSession` calls in `db.transaction()`.

`insertMessages(projectName, sessionId, entries)` — Accepts array of raw JSONL entry objects. For each entry, extracts `entry_type` (from `entry.type`), `message_id` (from `entry.message?.id || null`), `uuid` (from `entry.uuid || null`), `parent_uuid` (from `entry.parentUuid || null`), `timestamp` (from `entry.timestamp || null`). Stores `JSON.stringify(entry)` as `raw_json`. Uses `INSERT OR IGNORE INTO messages ...` (IGNORE on uuid conflict for dedup). Wrapped in a transaction for bulk performance.

`getSessionsByProject(projectName, limit, offset)` — `SELECT * FROM sessions WHERE project_name = ? AND is_junk = 0 ORDER BY last_activity DESC LIMIT ? OFFSET ?`. Returns array of session row objects. Also returns `total` via a separate COUNT query.

`getMessagesBySession(sessionId)` — `SELECT raw_json FROM messages WHERE session_id = ? ORDER BY timestamp ASC`. Returns array of parsed JSON objects (`JSON.parse(row.raw_json)`).

`getSessionMeta(sessionId)` — `SELECT * FROM sessions WHERE id = ?`. Returns session row or undefined.

`checkFreshness(sessionId, currentMtime, currentSize)` — Looks up session's `jsonl_mtime` and `jsonl_size`. Returns `true` if both match (cache is fresh), `false` otherwise (needs re-parse).

`invalidateSession(sessionId)` — `DELETE FROM messages WHERE session_id = ?` then `DELETE FROM sessions WHERE id = ?`. Wrapped in transaction.

`invalidateProject(projectName)` — `DELETE FROM messages WHERE project_name = ?` then `DELETE FROM sessions WHERE project_name = ?`. Wrapped in transaction.

`getStaleSessionsForProject(projectName, fileStats)` — Accepts a Map of `jsonlFile -> { mtimeMs, size }`. Queries all sessions for the project, compares each session's `jsonl_mtime` and `jsonl_size` against the provided stats. Returns array of session IDs that are stale (mtime or size mismatch).

`getUncachedFiles(projectName, allJsonlFiles)` — Accepts array of JSONL filenames. Queries distinct `jsonl_file` values for the project. Returns filenames not in cache.

`getAllCachedFiles(projectName)` — `SELECT DISTINCT jsonl_file FROM sessions WHERE project_name = ?`. Returns Set of filenames.

`close()` — Calls `this.db.close()`.

`stats()` — Returns `{ sessionCount, messageCount, dbSizeBytes }` using COUNT queries and `fs.statSync(dbPath).size`.

**Export:** Create singleton instance and export it: `const messageCache = new MessageCache(); export default messageCache; export { MessageCache };`

Key implementation details:
- All prepared statements stored on `this._stmts` object in constructor for reuse
- `insertMessages` MUST use a transaction — inserting 500+ rows without a transaction is 100x slower in SQLite
- The `uuid UNIQUE` constraint with `INSERT OR IGNORE` handles dedup naturally — re-importing the same JSONL entries is a no-op
- `checkFreshness` compares `jsonl_mtime` as a float (milliseconds since epoch from `stat.mtimeMs`) and `jsonl_size` as integer bytes
