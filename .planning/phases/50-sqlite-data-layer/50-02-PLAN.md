Create `server/cache/cache-warmer.js` that exports an async `warmCache()` function.

**Function: `warmCache(progressCallback = null)`**

This function runs in the background after server startup. It scans all JSONL project directories in `~/.claude/projects/`, checks which files need (re)indexing, and populates the MessageCache.

Implementation:

1. **Discover project directories:**
   ```javascript
   const claudeProjectsDir = path.join(os.homedir(), '.claude', 'projects');
   const entries = await fs.readdir(claudeProjectsDir, { withFileTypes: true });
   const projectDirs = entries.filter(e => e.isDirectory()).map(e => e.name);
   ```

2. **For each project directory:**
   - List all `.jsonl` files (excluding `agent-*.jsonl`)
   - Stat each file to get `mtimeMs` and `size`
   - Build a Map: `filename -> { mtimeMs, size }`
   - Call `messageCache.getUncachedFiles(projectName, allJsonlFilenames)` to find new files
   - Call `messageCache.getStaleSessionsForProject(projectName, fileStatsMap)` to find stale entries
   - Invalidate stale sessions
   - For uncached files: parse JSONL (reuse the same readline approach as `parseJsonlSessions`), then call `messageCache.upsertSession()` and `messageCache.insertMessages()` for each session found

3. **Yield between files** to avoid blocking the event loop:
   ```javascript
   // After processing each JSONL file:
   await new Promise(resolve => setImmediate(resolve));
   ```

4. **Progress logging:**
   ```javascript
   console.log(`[CACHE] Warming: ${processedFiles}/${totalFiles} files, ${cachedSessions} sessions cached`);
   ```
   Log at most every 10 files or every 5 seconds (whichever comes first) to avoid log spam.

5. **Error handling:** Individual file parse errors are logged and skipped (don't abort the entire warm). Wrap the outer function in try/catch and log any top-level errors.

6. **The parseJsonlForCache() helper** (internal to this module):
   - Reads a JSONL file line by line (same readline pattern as projects.js)
   - Collects all entries grouped by sessionId
   - For each session found, extracts metadata: `summary` (from latest 'summary' type entry), `messageCount`, `lastActivity` (latest timestamp), `cwd` (from first entry with cwd), `lastUserMessage` (last 'user' type message text), `lastAssistantMessage` (last 'assistant' type message text)
   - Returns `{ sessions: Map<sessionId, sessionMeta>, entries: Entry[] }`
   - This is intentionally separate from `parseJsonlSessions` in projects.js because it has a different output shape (optimized for cache population, not for REST response)

7. **Export:**
   ```javascript
   export { warmCache };
   ```

Key constraints:
- MUST NOT import from `projects.js` (avoid circular dependency — cache-warmer does its own JSONL parsing)
- MUST use `setImmediate` between files to yield to the event loop
- MUST handle `ENOENT` gracefully (directory may not exist on fresh install)
- MUST complete within 30 seconds for 383 files / 646MB (the loom project dataset)
- Log total time elapsed at completion: `[CACHE] Warming complete: ${sessions} sessions, ${messages} messages in ${elapsed}s`
