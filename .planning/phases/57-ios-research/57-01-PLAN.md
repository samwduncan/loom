## Executive Summary
- One paragraph: Capacitor 7.6.1 wraps the existing Vite+React SPA in a WKWebView. Two deployment models exist. Tailscale DNS works theoretically (HIGH confidence). No Mac available so on-device testing is deferred. Effort estimate for each milestone.

## Deployment Models
Two subsections:

### Model A: Bundled Assets (App Store Path)
- How it works: web build embedded in app binary, API calls to remote 100.86.4.57:5555
- Pros: offline-capable UI, App Store compliant, fast initial load
- Cons: requires rebuild/resubmit for UI changes, needs API base URL to be configurable (currently relative `/api` paths), CORS headers on backend
- Required code changes: (1) API base URL abstraction in frontend (currently uses relative paths via Vite proxy), (2) CORS headers on Express backend for `capacitor://localhost` origin, (3) Explicit WebSocket URL construction (`ws://100.86.4.57:5555/ws` instead of relative `/ws`)
- Effort: 2-3 days with Mac + Apple Developer Account ($99/yr)

### Model B: Remote server.url (Personal Use Path)
- How it works: WKWebView loads http://100.86.4.57:5184, all requests proxied through Vite
- Pros: zero code changes, zero rebuild for UI updates, identical to browser
- Cons: Apple rejects under Guideline 4.2, requires active Tailscale VPN, no offline
- Effort: 2-4 hours with Mac

## Tailscale DNS from WKWebView (IOS-02)
- Verdict: HIGH confidence it works
- Mechanism: Tailscale iOS uses NetworkExtension system-wide VPN. WKWebView networking runs in a separate system process. System VPN applies to all processes including WKWebView networking daemon. Traffic to 100.x.y.z routes through Tailscale tunnel.
- Caveats: (1) ATS requires NSAllowsArbitraryLoads for HTTP+IP (Capacitor sets this by default), (2) MagicDNS `.ts.net` with HTTPS is cleaner but untested from WKWebView, (3) VPN must be active -- silent failure otherwise, (4) WebSocket `ws://` works through VPN tunnel
- What cannot be tested from Linux: actual on-device verification, ATS enforcement, WS latency on cellular, MagicDNS hostname timing
- Recommended testing protocol: 7 steps (install Tailscale on iPhone, Safari test, Capacitor build, WKWebView test, WS test, disconnected test, MagicDNS test)

- Guideline 4.2 (minimum functionality): bundled assets required, push notifications recommended, native splash screen needed
- Must-haves for approval: bundle web assets, push notifications, native splash/icon, offline error states, SPA navigation (already satisfied)
- Nice-to-haves: Face ID/Touch ID, haptics, share extension
- Risk: Apple may still reject as "wrapped website" -- mitigate with native features

|-----------|--------|-------------|-------------|

| App Store minimum viable | 2-3 days | Mac, Apple Dev Account ($99/yr), push notification server | Add native features to pass review |
| Production quality | 1-2 weeks | All above + proper offline handling | Full native feature set |
