  <behavior>
    - convertStreamingMarkdown("**bold**") returns HTML containing `<strong>bold</strong>`
    - convertStreamingMarkdown("*italic*") returns HTML containing `<em>italic</em>`
    - convertStreamingMarkdown("`code`") returns HTML containing `<code>code</code>`
    - convertStreamingMarkdown("# Heading") returns HTML containing `<h1>Heading</h1>`
    - convertStreamingMarkdown("## H2\n### H3") returns correct heading levels
    - convertStreamingMarkdown("- item1\n- item2") returns `<ul>` with `<li>` children
    - convertStreamingMarkdown("1. first\n2. second") returns `<ol>` with `<li>` children
    - convertStreamingMarkdown("> quote") returns `<blockquote>` with content
    - convertStreamingMarkdown("```js\nconst x = 1;\n```") returns `<pre>` with `<div>` wrapper and language label
    - convertStreamingMarkdown("```js\nconst x = 1;") (unclosed fence) returns a code block container (synthetic close injected)
    - convertStreamingMarkdown("text\n| col1 | col2 |\n| --- | --- |\n| a | b |") with trailing pipe and no empty line after: table portion renders as plain text
    - convertStreamingMarkdown('<script>alert("xss")</script>') strips the script tag entirely
    - convertStreamingMarkdown('<img onerror="alert(1)" src="x">') strips the img tag
    - convertStreamingMarkdown("normal paragraph\n\nanother paragraph") wraps in `<p>` tags
    - Allowlist enforcement: only `<strong>`, `<em>`, `<code>`, `<a>`, `<p>`, `<br>`, `<ul>`, `<ol>`, `<li>`, `<h1>`-`<h6>`, `<pre>`, `<div>` survive sanitization
  </behavior>
  <action>
    1. Install DOMPurify: `cd /home/swd/loom/src && npm install dompurify @types/dompurify`

    2. Create `src/src/lib/streaming-markdown.ts`:
       - Single exported pure function `convertStreamingMarkdown(raw: string): string`
       - Processing order (critical for correctness):
         a. Extract code fences first: find all ```...``` blocks, replace with placeholders. For unclosed fences (odd count of ```), inject synthetic closing fence.
         b. Extract inline code: replace `\`...\`` with placeholders
         c. Convert headings: `^#{1,6}\s+(.+)$` -> `<hN>...</hN>`
         d. Convert blockquotes: `^>\s?(.*)` -> `<blockquote>...</blockquote>` (handle multi-line)
         e. Convert lists: detect `^[-*]\s` for ul and `^\d+\.\s` for ol, wrap in `<ul>`/`<ol>` with `<li>` items
         f. Convert bold: `**...**` and `__...__` -> `<strong>`
         g. Convert italic: `*...*` and `_..._` -> `<em>` (using negative lookbehind to avoid bold conflicts)
         h. Convert links: `[text](url)` -> `<a href="url">text</a>`
         i. Convert paragraphs: double newlines create `<p>` wrappers
         j. Convert line breaks: single newlines within paragraphs -> `<br>`
         k. Restore inline code placeholders
         l. Restore code fence placeholders as `<pre><div class="streaming-code-block"><div class="streaming-code-lang">{lang}</div><code>{content}</code></div></pre>`
       - Incomplete table detection: if last content section starts with `|` and no empty line follows, leave as raw text (don't parse)
       - Sanitize with DOMPurify: `DOMPurify.sanitize(html, { ALLOWED_TAGS, ALLOWED_ATTR: ['href', 'target', 'rel', 'class'], KEEP_CONTENT: true })`
       - ALLOWED_TAGS: `['strong', 'em', 'code', 'a', 'p', 'br', 'ul', 'ol', 'li', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6', 'pre', 'div', 'blockquote']`

    3. Create `src/src/lib/streaming-markdown.test.ts` with tests for all behaviors listed above, plus edge cases:
       - Mixed formatting: `**bold *and italic* text**`
       - Code containing asterisks: `` `**not bold**` ``
       - Empty input returns empty string
       - Very large input (10KB+) completes without throwing

    Note: DOMPurify works in jsdom (Vitest). Import normally -- it auto-detects the environment.

             converterFailedRef.current = true;
           }
