**Step 4: Create `.husky/pre-commit` hook script:**
Replace the default content `npx husky init` created with:

```bash
#!/usr/bin/env sh

# Only run frontend checks if src/ files are staged
STAGED_SRC=$(git diff --cached --name-only --diff-filter=ACMR | grep '^src/' || true)

if [ -z "$STAGED_SRC" ]; then
  echo "No src/ files staged -- skipping frontend checks"
  exit 0
fi

echo "=== Pre-commit: Frontend checks ==="

# Step 1: lint-staged (auto-fixes fixable issues and re-stages)
echo "[1/3] Running lint-staged..."
cd src
npx lint-staged --concurrent false
LINT_EXIT=$?
if [ $LINT_EXIT -ne 0 ]; then
  echo "FAILED: lint-staged found unfixable errors"
  exit 1
fi

# Step 2: Full project typecheck (catches cross-file breakage)
echo "[2/3] Running typecheck..."
npx tsc -b --noEmit
TSC_EXIT=$?
if [ $TSC_EXIT -ne 0 ]; then
  echo "FAILED: TypeScript errors found"
  exit 1
fi

# Step 3: Affected tests only (no coverage -- coverage is for CI/gate checks)
# Uses `vitest related --run` to only run tests affected by staged files.
# This keeps the hook fast (<30s) as the test suite grows.
# Full coverage check: use `npm run test:coverage` or CI pipeline.
echo "[3/3] Running affected tests..."
npx vitest related --run $(git diff --cached --name-only --diff-filter=ACMR | grep '^src/src/.*\.\(ts\|tsx\)$' | sed 's|^src/||')
TEST_EXIT=$?
if [ $TEST_EXIT -ne 0 ]; then
  echo "FAILED: Tests failed"
  exit 1
fi

echo "=== All checks passed ==="
```

**IMPORTANT (Issue #3 fix -- pre-commit scalability):** The hook uses `npx vitest related --run` (affected tests only, NO coverage) instead of `npx vitest run --coverage`. This ensures the hook stays under the 30-second mandate as the codebase grows. Full `vitest run --coverage` is reserved for:
- `npm run test:coverage` (manual runs)
- CI/gate checks (`/gsd:verify-work`)
- GSD executor agents' pre-commit validation

The `vitest related` command accepts file paths and only runs tests that import (directly or transitively) those files. The `sed 's|^src/||'` strips the leading `src/` from paths because vitest runs from inside the `src/` directory.

Make sure the file is executable: `chmod +x .husky/pre-commit`
