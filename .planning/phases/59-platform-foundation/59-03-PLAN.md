1. Update the cors() middleware origin array (currently at line 339-341). Change from:
   ```javascript
   app.use(cors({
       origin: ['https://samsara.tailad2401.ts.net:5443', 'http://100.86.4.57:5184', 'http://localhost:5184'],
   }));
   ```
   To a function-based origin that logs unrecognized origins:
   ```javascript
   const ALLOWED_ORIGINS = [
       'https://samsara.tailad2401.ts.net:5443',
       'http://100.86.4.57:5184',
       'http://localhost:5184',
       'capacitor://localhost',
   ];

   app.use(cors({
       origin: (origin, callback) => {
           // Allow requests with no origin (same-origin, curl, etc.)
           if (!origin) return callback(null, true);
           if (ALLOWED_ORIGINS.includes(origin)) return callback(null, true);
           // Debug: log unrecognized origins during development (remove before v2.2)
           console.warn('[CORS] Rejected unrecognized origin:', origin);
           callback(new Error(`CORS: origin ${origin} not allowed`));
       },
   }));
   ```

   This approach (per D-18):
   - Adds `capacitor://localhost` as exact string (Pitfall 4 -- no trailing slash, no wildcards)
   - Logs rejected origins for debugging during development
   - Uses a function instead of array to enable the logging behavior
   - Passes `null, true` for no-origin requests (server-to-server, curl, same-origin fetch)
