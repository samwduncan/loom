**IMPORTANT: This task requires backend investigation first.**

1. **Investigate the Claude SDK interactive prompt mechanism.** In `server/claude-sdk.js`, examine how interactive prompts work. The Claude SDK likely uses stdin to accept user input for interactive prompts. Check:
   - How does the running Claude process receive the user's selection?
   - Is there a `process.stdin.write()` or similar mechanism?
   - Does the SDK expose a callback or event for sending interactive responses?

2. **If stdin-based:** Add a WebSocket handler in `server/routes/agent.js` that accepts interactive prompt responses:
   ```javascript
   // In the WebSocket message handler
   case 'interactive-prompt-response':
     // data.sessionId, data.selectedOption (e.g., "1" for option 1)
     const session = getSession(data.sessionId);
     if (session?.claudeProcess?.stdin) {
       session.claudeProcess.stdin.write(data.selectedOption + '\n');
     }
     break;
   ```

3. **If SDK callback-based:** Wire the callback to accept WebSocket responses. The approach depends on how the SDK exposes the prompt interaction.

4. **Frontend: Enable option buttons** in MessageComponent.tsx:
   - Remove `disabled` and `cursor-not-allowed opacity-75` from option buttons
   - **Socket access:** Import `useWebSocket` from `../../../../contexts/WebSocketContext`. The app already wraps all components in `WebSocketProvider` (see `App.tsx`). Call `const { sendMessage } = useWebSocket();` at the top of the MessageComponent function body. This hook is already used by `useChatSessionState` and `useChatComposerState` for permission decisions and session management -- the same pattern applies here.
   - Add onClick handler that sends WebSocket message via the context's `sendMessage`:
     ```tsx
     const { sendMessage: wsSendMessage } = useWebSocket();
     // ... inside render:
     onClick={() => {
       wsSendMessage({
         type: 'interactive-prompt-response',
         sessionId: currentSessionId,
         selectedOption: option.number,
       });
       setPromptResponded(true);
     }}
     ```
   - Add local state `const [promptResponded, setPromptResponded] = useState(false);` to track that a response was sent from this client
   - After clicking, disable all buttons and show "Response sent" instead of pulse indicator
   - Note: `currentSessionId` must be passed as a prop to MessageComponent (add to `TurnBlockMessageProps` interface in TurnBlock.tsx, sourced from `ChatMessagesPane`'s `currentSessionId` prop)

5. **Race condition handling:** Both CLI and web UI can respond. The first response should win. After the CLI or web UI responds:
   - The server should broadcast a `claude-interactive-prompt-resolved` event
   - The frontend should update the prompt card to show the selected option and disable further interaction

6. **If backend integration is infeasible in this phase:** Complete the visual restyle (Task 1) but leave buttons disabled. Add a comment in the code noting the websocket integration is deferred. Update CONTEXT.md deferred section.
