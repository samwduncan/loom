Specific integration points to verify and fix:

1. **Session list -> Chat navigation:**

   - Composer calls `getWsClient().send({ type: 'claude-command', command: text, options: { sessionId: id, projectPath } })`
   - Verify projectPath is available on the chat screen (from route params or from useSessions hook via session data)

3. **Streaming state -> Session list indicators:**
   - When a session is actively streaming, its SessionItem should show the pulsing accent dot (per D-11)
   - This requires the session list to subscribe to useStreamStore or have a way to know which session is streaming
   - Add: check if `activeSessionId === session.id && isStreaming` in SessionItem

4. **Drawer index screen:**
   - The (drawer)/index.tsx is the "main content" that shows when drawer is open or closed
   - It should either:
     (a) Show the last active chat (redirect to chat/[id] if activeSessionId exists), OR
     (b) Show EmptyChat when no active session
   - Verify this routing works with Expo Router's drawer+stack groups

5. **Root layout integration:**
   - Verify the auth gate -> WebSocket init -> ConnectionBanner -> Drawer flow works end-to-end
   - Verify that when the user authenticates via AuthPrompt, the session list immediately loads (fetchSessions called after auth)
   - Add: call fetchSessions in the root layout after authentication, or ensure SessionList fetches on mount

6. **AppState lifecycle:**
   - Verify that backgrounding triggers the 30s disconnect timer (in websocket-init.ts)
   - Verify that foregrounding clears the timer and reconnects if disconnected
   - Verify ConnectionBanner shows during disconnected state and dismisses on reconnect

7. **Soul doc anti-pattern audit:**
   - Anti-pattern #2 (no instant state changes): verify all state transitions animate
   - Anti-pattern #11 (no silent interactions): verify every interactive element has press feedback
   - Anti-pattern #13 (no loading spinners without context): verify loading states have descriptive text
   - Anti-pattern #14 (no stale state after backgrounding): verify state syncs on foreground return
