- `getWsClient()`: returns current wsClient
- `initializeWebSocket()`: async function that:
  1. Guards against double-init (isInitialized check, per Pitfall 6)

     - onThinkingBlock -> useStreamStore.getState().setThinkingBlock

     - onToolResult -> useStreamStore.getState().completeToolCall

     - onStreamEnd -> useStreamStore.getState().endStream + useTimelineStore.getState().refreshSession
     - onError -> useConnectionStore.getState().setError
     - onSessionCreated -> useTimelineStore.getState().setActiveSession
     - onActiveSessions -> useTimelineStore.getState().setActiveSessions
  4. Configures wsClient.configure({ onMessage, onStateChange })

     - onStateChange: maps to useConnectionStore.getState().updateStatus
  5. AppState.addEventListener('change', handler):
     - 'active': clear backgroundTimer, if disconnected call wsClient.tryReconnect()
     - 'background': start 30s timer that calls wsClient.disconnect()
     - 'inactive': no-op
  6. Reads token from nativeAuthProvider.getToken(), if exists calls wsClient.connect(token)
- `disconnectWebSocket()`: disconnect and reset state

Note: DO NOT use window.dispatchEvent or localStorage -- use Zustand actions directly. DO NOT use import.meta.env -- URLs come from platform.ts.
