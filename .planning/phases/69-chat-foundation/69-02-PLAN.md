**Step 1: Create mobile/hooks/useAuth.ts** -- manages auth state:
```typescript
interface UseAuthReturn {
  isAuthenticated: boolean;
  isLoading: boolean;
  token: string | null;
  login: (token: string) => Promise<boolean>;  // validates token against /api/auth/user
  logout: () => void;
  checkAuth: () => void;  // reads from Keychain on app start
}
```
- `checkAuth()`: reads nativeAuthProvider.getToken(). If token exists, sets isAuthenticated=true.
- `login(token)`: calls nativeAuthProvider.setToken(token), then validates by fetching `GET /api/auth/user` with the token. If 200, sets isAuthenticated=true. If fails, clears token, returns false, shows error state.
- `logout()`: calls nativeAuthProvider.clearToken(), sets isAuthenticated=false.
- Uses React useState for isAuthenticated, isLoading.
- Per D-07: auto-connect pattern. Token prompt only on first launch or invalid token.

**Step 2: Create mobile/hooks/useConnection.ts** -- subscribes to Zustand connection store:
```typescript
interface UseConnectionReturn {
  status: WsConnectionState;  // 'disconnected' | 'connecting' | 'connected' | 'reconnecting'
  isConnected: boolean;
  isReconnecting: boolean;
  error: string | null;
}
```
- Reads from useConnectionStore with shallow equality selector
- Derives isConnected from status === 'connected'
- Derives isReconnecting from status === 'reconnecting'
- Returns error from connection store state

**Rewrite mobile/app/_layout.tsx** to be the app's root provider and lifecycle manager.

The current root layout is minimal (just GestureHandlerRootView + Drawer). It needs to become the central orchestration point.

New structure:
```tsx
import '../global.css';
import { useEffect, useState } from 'react';
import { View } from 'react-native';
import { GestureHandlerRootView } from 'react-native-gesture-handler';
import { KeyboardProvider } from 'react-native-keyboard-controller';
import { Drawer } from 'expo-router/drawer';
import { AuthPrompt } from '../components/connection/AuthPrompt';
import { ConnectionBanner } from '../components/connection/ConnectionBanner';
import { useAuth } from '../hooks/useAuth';
import { useConnection } from '../hooks/useConnection';
import { initializeWebSocket } from '../lib/websocket-init';

export default function RootLayout() {
  const { isAuthenticated, isLoading, checkAuth, login, logout } = useAuth();
  const { status } = useConnection();

  // Check for existing token on app start
  useEffect(() => {
    checkAuth();
  }, []);

  // Initialize WebSocket when authenticated
  useEffect(() => {
    if (isAuthenticated) {
      initializeWebSocket();
    }
  }, [isAuthenticated]);

  return (
    <GestureHandlerRootView style={{ flex: 1, backgroundColor: 'rgb(46, 42, 40)' }}>
      <KeyboardProvider>
        {!isAuthenticated && !isLoading ? (
          <AuthPrompt onLogin={login} />
        ) : (
          <View style={{ flex: 1 }}>
            <ConnectionBanner />
            <Drawer
              screenOptions={{
                headerShown: false,
                drawerStyle: {
                  backgroundColor: 'rgb(38, 35, 33)',
                  width: 300,
                },
                sceneStyle: {
                  backgroundColor: 'rgb(46, 42, 40)',
                },
              }}
            >
              <Drawer.Screen name="(drawer)" options={{ headerShown: false }} />
              <Drawer.Screen
                name="(stack)"
                options={{ drawerItemStyle: { display: 'none' } }}
              />
            </Drawer>
          </View>
        )}
      </KeyboardProvider>
    </GestureHandlerRootView>
  );
}
```

Key requirements:
1. **KeyboardProvider wraps everything** -- required by react-native-keyboard-controller for KeyboardStickyView in the composer (D-16). Must be near root, above any screen that uses keyboard avoidance.
2. **Auth gate** -- if not authenticated and not loading, show AuthPrompt. Otherwise show main app. Per D-09: no separate onboarding.
3. **WebSocket init** -- called once when isAuthenticated becomes true. The initializeWebSocket() function has its own double-init guard.
4. **ConnectionBanner** -- rendered above the Drawer, visible on all screens. Shows/hides based on connection store state.
5. **Drawer width** -- 300px per Soul doc (80% of screen, max 320px). Drawer bg: surface-sunken.
6. **No StatusBar component yet** -- React Native manages status bar. Light content on dark backgrounds is the default for dark mode apps.

Note: AppState lifecycle is handled inside websocket-init.ts (registered during initializeWebSocket), NOT in the root layout. The root layout only handles auth gating and WebSocket initialization trigger.
