**Step 1: Create mobile/hooks/useAuth.ts** -- manages auth state:
```typescript
interface UseAuthReturn {
  isAuthenticated: boolean;
  isLoading: boolean;
  token: string | null;
  login: (token: string) => Promise<boolean>;  // validates token against /api/auth/user
  logout: () => void;
  checkAuth: () => void;  // reads from Keychain on app start
}
```

- `logout()`: calls nativeAuthProvider.clearToken(), sets isAuthenticated=false.
- Uses React useState for isAuthenticated, isLoading (isLoading starts TRUE).

CRITICAL FIX per S-7 (Dual Drawer nesting): Root `_layout.tsx` must use `<Slot />` (or `<Stack>`), NOT `<Drawer>`. The Drawer navigator belongs EXCLUSIVELY in `(drawer)/_layout.tsx`. Two nested Drawer navigators = runtime crash in Expo Router.

CRITICAL FIX per A-6 (blank flash during checkAuth): Implement 3-way render: `isLoading ? <SplashView/> : !isAuthenticated ? <AuthPrompt/> : <MainApp/>`.

New structure:
```tsx
import '../global.css';
import { useEffect } from 'react';
import { View, ActivityIndicator } from 'react-native';
import { GestureHandlerRootView } from 'react-native-gesture-handler';
import { KeyboardProvider } from 'react-native-keyboard-controller';
import { Slot } from 'expo-router';
import { AuthPrompt } from '../components/connection/AuthPrompt';
import { ConnectionBanner } from '../components/connection/ConnectionBanner';
import { useAuth } from '../hooks/useAuth';
import { initializeWebSocket } from '../lib/websocket-init';
import { SURFACE } from '../lib/colors';

export default function RootLayout() {
  const { isAuthenticated, isLoading, checkAuth, login, logout } = useAuth();

  // Check for existing token on app start
  useEffect(() => {
    checkAuth();
  }, []);

  // Initialize WebSocket when authenticated
  useEffect(() => {
    if (isAuthenticated) {
      initializeWebSocket();
    }
  }, [isAuthenticated]);

  return (
    <GestureHandlerRootView style={{ flex: 1, backgroundColor: SURFACE.base }}>
      <KeyboardProvider>
        {isLoading ? (
          // Splash state -- prevents blank flash while reading token from Keychain
          <View style={{ flex: 1, backgroundColor: SURFACE.base, justifyContent: 'center', alignItems: 'center' }}>
            <ActivityIndicator color={SURFACE.overlay} />
          </View>
        ) : !isAuthenticated ? (
          <AuthPrompt onLogin={login} />
        ) : (
          <View style={{ flex: 1 }}>
            <ConnectionBanner />
            <Slot />
          </View>
        )}
      </KeyboardProvider>
    </GestureHandlerRootView>
  );
}
```

Key requirements:
1. **`<Slot />`** NOT `<Drawer>` -- Root layout renders child routes via Slot. The Drawer navigator lives in `(drawer)/_layout.tsx`. This prevents the dual drawer nesting crash (S-7).
2. **3-way render per A-6:** isLoading (splash) -> !isAuthenticated (AuthPrompt) -> authenticated (main app). No blank flash.
3. **KeyboardProvider wraps everything** -- required by react-native-keyboard-controller for KeyboardStickyView in the composer (D-16). Must be near root, above any screen that uses keyboard avoidance.
4. **WebSocket init** -- called once when isAuthenticated becomes true. The initializeWebSocket() function has its own double-init guard.
5. **ConnectionBanner** -- rendered above the Slot, visible on all screens. Shows/hides based on connection store state (with hasConnectedOnce guard from Task 1).
6. **No StatusBar component yet** -- React Native manages status bar. Light content on dark backgrounds is the default for dark mode apps.

Note: AppState lifecycle is handled inside websocket-init.ts (registered during initializeWebSocket), NOT in the root layout. The root layout only handles auth gating and WebSocket initialization trigger.
