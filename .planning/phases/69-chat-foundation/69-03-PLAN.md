**Step 1: Create mobile/hooks/useSessions.ts** -- session data management:
```typescript
interface UseSessions {
  projects: ProjectWithSessions[];
  pinnedSessions: SessionData[];
  isLoading: boolean;
  error: string | null;
  activeSessionId: string | null;
  searchQuery: string;
  setSearchQuery: (q: string) => void;
  fetchSessions: () => Promise<void>;
  createSession: (projectName: string) => void;  // sends claude-command via WS
  deleteSession: (projectName: string, sessionId: string) => Promise<void>;
  setActiveSession: (sessionId: string) => void;
}

interface ProjectWithSessions {
  name: string;
  displayName: string;
  provider: string;
  sessions: SessionData[];
}

interface SessionData {
  id: string;
  title: string;
  projectName: string;
  provider: string;
  lastActivity: string;
  isPinned: boolean;
  isStreaming: boolean;
}
```
- Create an API client instance using createApiClient({ auth: nativeAuthProvider, resolveUrl: resolveApiUrl })
- `fetchSessions()`: GET /api/projects, then for each project GET /api/projects/:name/sessions?limit=50. Transform into ProjectWithSessions[] with sessions sorted by lastActivity desc.
- `deleteSession()`: DELETE /api/projects/:name/sessions/:id, then refetch
- `createSession()`: calls getWsClient().send({ type: 'claude-command', command: '', options: { projectPath } }) -- the WebSocket will create a new session on the backend
- `setActiveSession()`: updates timeline store activeSessionId
- Search filtering: filter sessions where title includes searchQuery (case-insensitive)
- Pinned sessions: stored in MMKV as JSON array of session IDs (separate from backend). Read on mount, write on pin/unpin.
- Returns loading skeletons state for Soul doc anti-pattern #13
