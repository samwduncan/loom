# Phase 75 — UI Design Contract

> Visual and interaction contract for the Chat Shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This contract builds on the established theme system from Phase 74 (mobile/theme/) and the NATIVE-APP-SOUL.md visual authority.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native -- shadcn not applicable) |
| Preset | not applicable |
| Component library | Custom components with `createStyles(theme)` pattern |
| Icon library | lucide-react-native (primary), SF Symbols via expo-symbols (system UI) |
| Font | Inter (UI: Regular 400, SemiBold 600), JetBrains Mono (code: Regular 400) |

**Existing theme:** `mobile/theme/theme.ts` -- singleton LoomTheme with colors, typography, spacing, springs, shadows, radii. All Phase 75 components consume this via `createStyles((t) => ({...}))`.

---

## Spacing Scale

Declared values (multiples of 4). Already implemented in `theme.spacing`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tool chip internal padding |
| sm | 8px | Same-role message spacing, tool card group spacing, tool chip gap |
| md | 16px | Default element spacing, screen horizontal margin (px-4), message horizontal padding |
| lg | 24px | Turn spacing (user-to-assistant), section padding, provider avatar size |
| xl | 32px | Layout gaps, drawer header spacing |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **44px** minimum touch target on all interactive elements (send button, approve/deny, copy code, tool chips). Use `hitSlop` on visually smaller elements. Source: Soul doc anti-pattern #4.
- **36px** send button diameter (visual size; touch target remains 44px via hitSlop). Source: Soul doc Composer spec.
- **56px** minimum session item height in drawer. Source: Soul doc Session List spec.

## Color

All values from `mobile/lib/colors.ts` and `mobile/theme/theme.ts`. No overrides needed.

### Surface Hierarchy (60/30/10 split)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgb(44, 40, 38)` surface-base | Chat background, scroll areas, default canvas |
| Secondary (30%) | `rgb(64, 60, 57)` surface-raised / `rgb(28, 26, 24)` surface-sunken | User message bubbles, composer (raised); drawer background, code blocks (sunken) |
| Accent (10%) | `rgb(196, 108, 88)` | See reserved-for list below |
| Destructive | `rgb(210, 112, 88)` | Swipe-to-delete action surface, stop button during streaming, deny action emphasis, error states |
| Success | `rgb(82, 175, 108)` | Tool call completion status dot, connection status (connected) |

### Accent Reserved-For List

Accent `rgb(196, 108, 88)` is used ONLY on these elements:

1. Send button background (when text present, with glow shadow)
2. Active session indicator dot in drawer (pulsing during streaming)
3. Approve button background on permission cards
4. Connection status dot (connected state)
5. Streaming indicator line (2px pulsing line at bottom of message area)
6. Scroll-to-bottom pill text
7. "New Chat" button in empty session list
8. Code block "Copy" button text
9. Tool chip status dot (running state)
10. Search match highlight background (accent at 0.15 opacity)

### Text Colors

| Token | Value | Usage |
|-------|-------|-------|
| text-primary | `rgb(230, 222, 216)` | Message text, headings, button labels |
| text-secondary | `rgb(191, 186, 182)` | Session subtitle (drawer), tool card body, secondary info |
| text-muted | `rgb(148, 144, 141)` | Timestamps, placeholders, "Thought for Xs" label, caption text, thinking block text |

### Border Colors

| Token | Value | Usage |
|-------|-------|-------|
| border-subtle | `rgba(255,255,255,0.07)` | Default card borders, glass surface border, code block container |
| border-interactive | `rgba(255,255,255,0.34)` | Active tool card border (running), focused input border, active state |

### Dynamic Color States

| State | Background Shift | Accent Behavior | Duration |
|-------|-----------------|-----------------|----------|
| Idle | `rgb(46, 42, 40)` (IDLE_BG) | Static | -- |
| Streaming | `rgb(48, 43, 40)` (STREAMING_BG, +3 hue warmth) | Opacity pulse 0.6-1.0, 2.5s sinusoidal | 500ms transition in, 500ms out |
| Error | `rgb(44, 42, 42)` (ERROR_BG, cooler) | Destructive replaces accent temporarily | 500ms transition |
| Permission | No background shift | Glow shadow on permission card, UI dims to 0.95 opacity | 200ms dim |
| Tool executing | No background shift | Shimmer sweep on active tool card, 2s cycle | Continuous during execution |

All color transitions use `withTiming(value, { duration: 500, easing: Easing.out(Easing.cubic) })`. Not springs.

## Interaction Contracts

### Haptic Pairing (from Soul doc)

| Event | Haptic Type | Timing |
|-------|-------------|--------|
| Message sent | Impact Light | On send button press |
| Tool chip tap | Selection | On chip press |
| Tool card complete | Success Notification | On status change to complete |
| Permission request appears | Warning Notification | On appearance animation start |
| Permission approved | Success Notification | On approve button press |
| Permission denied | Impact Light | On deny button press |
| Swipe delete threshold | Selection | When finger crosses action threshold |
| Code copied | Impact Light | On copy button press |
| Drawer snap | Impact Medium | When drawer reaches final position |
| Session item press | Selection | On press |
| Scroll-to-bottom tap | Selection | On tap |
| Send button press | Impact Light | On press |
| Stop button press | Impact Medium | On press |

### Spring Usage Map

| Spring | Config | Components |
|--------|--------|------------|
| Micro | 18/220/0.8 (~150ms) | Send button press, tool chip press, session item press, approve/deny press |
| Standard | 20/150/1.0 (~250ms) | User bubble entrance, search expand/collapse, swipe-delete reveal, undo toast, copied toast, scroll pill, multi-line composer height |
| Navigation | 22/130/1.0 (~300ms) | Bottom sheet presentation, screen push/pop |
| Drawer | 20/100/1.0 (~350ms) | Sidebar open/close |
| Expand | 18/90/1.0 (~400ms) | Thinking block expand/collapse, code block show more/less |
| Dramatic | 15/70/1.2 (~500ms) | Permission card entrance |

### Motion Rules

1. Every interactive element responds to press with Micro spring scale (0.97 press, 1.0 release).
2. Every appearance uses at minimum Standard spring with opacity + translateY.
3. Streaming text does NOT animate per-character. Text appends, scroll follows.
4. All springs respect `AccessibilityInfo.isReduceMotionEnabled` -- replace with instant transitions.
5. Color transitions use `withTiming` (500ms, cubic ease-out), NOT springs.
6. Session list items stagger-animate on drawer open: 30ms delay per item, max 10 animated, rest appear instantly.
