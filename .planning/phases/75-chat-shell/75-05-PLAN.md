    **AR fix #3: Stale closure on 5s fallback timeout.**
    The original plan had a `setTimeout(() => { if (composerState === 'sending') setComposerState('idle') }, 5000)` inside handleSend. This captures stale `composerState` via closure -- if isStreaming fires before 5s (common), the timeout still fires mid-stream and resets to idle.

    **Fix:** Use a ref to track the fallback, and clear it when transitioning to active:
    ```typescript
    const fallbackTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

    const handleSend = useCallback(() => {
      if (!text.trim() || composerState !== 'idle') return;
      Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
      setComposerState('sending');
      const wsClient = getWsClient();
      if (!wsClient) { setComposerState('idle'); return; }
      const options: any = { projectPath };
      if (sessionId && !sessionId.startsWith('stub-')) options.sessionId = sessionId;
      wsClient.send({ type: 'claude-command', command: text.trim(), options });
      setText('');

      // Fallback: if stream never starts within 5s, return to idle
      fallbackTimerRef.current = setTimeout(() => {
        // Check composerState via functional updater to avoid stale closure
        setComposerState((prev) => prev === 'sending' ? 'idle' : prev);
      }, 5000);
    }, [text, composerState, sessionId, projectPath]);

    // Clear fallback timer when transitioning to active
    useEffect(() => {
      if (isStreaming) {
        if (fallbackTimerRef.current) {
          clearTimeout(fallbackTimerRef.current);
          fallbackTimerRef.current = null;
        }
        setComposerState('active');
      } else if (composerState === 'active') {
        setComposerState('idle');
      }
    }, [isStreaming]); // composerState intentionally omitted to avoid loop
    ```
    Key changes from original:
    1. `setComposerState((prev) => prev === 'sending' ? 'idle' : prev)` uses functional updater -- always reads latest state, no stale closure.
    2. `fallbackTimerRef` is cleared when `isStreaming` transitions to true -- timer never fires mid-stream.
