    - replaceSessionId preserves all session data (messages, title, metadata, timestamps)
    - replaceSessionId is a no-op if oldId not found
    - onSessionCreated uses replaceSessionId instead of add+copy+remove pattern
    - Stale stub sessions (older than 30 seconds) are cleaned up
    - onActiveSessions callback updates a set of streaming session IDs (no longer a no-op)
  </behavior>
  <action>
    1. **timeline.ts** -- Add `replaceSessionId` action to TimelineState interface and implementation:
       ```typescript
       replaceSessionId: (oldId: string, newId: string) => {
         set((state) => {
           const session = state.sessions.find((s) => s.id === oldId);
           if (session) {
             session.id = newId;
           }
           if (state.activeSessionId === oldId) {
             state.activeSessionId = newId;
           }
         });
       },
       ```
       This uses Immer's mutable syntax (already configured). No partialize or merge changes needed -- the persist layer already handles sessions by their id field.

    2. **websocket-init.ts** -- Refactor `onSessionCreated` to use `replaceSessionId`:
       - Replace the add+copy+remove pattern with a single `timelineStore().replaceSessionId(stubSession.id, sid)` call
       - Keep the `window.history.replaceState` URL update
       - Add draft key migration: read `loom-composer-drafts` from localStorage, if a key exists for the stub ID, rename it to the real ID
       - Keep the fallback path (no stub found = add session normally)

    3. **websocket-init.ts** -- Wire `onActiveSessions` callback:
       - Parse the `{ claude: string[], codex: string[], gemini: string[] }` response
       - Flatten all session IDs into a single array
       - Store in a module-scoped `Set<string>` called `activeStreamingSessions`
       - Export a getter `getActiveStreamingSessions(): ReadonlySet<string>` for SessionList to consume
       - On reconnect, if `isCurrentlyStreaming` was true but the active session is not in the new active set, call `streamStore().endStream()` to clear stale state

    4. **websocket-init.ts** -- Add stub cleanup: In `onSessionCreated`, after successful reconciliation, schedule a `setTimeout(30_000)` that checks for any remaining `stub-` prefixed sessions and removes them. Store the timer ID in a module-scoped variable and clear it on subsequent `onSessionCreated` calls.

    5. **timeline.test.ts** -- Add tests for `replaceSessionId`:
       - Swaps session ID correctly
       - Updates activeSessionId if it matched the old ID
       - Preserves messages and metadata
       - No-op for non-existent oldId

    6. **websocket-init.test.ts** -- Add tests for hardened reconciliation:

       - onActiveSessions populates the active sessions set
