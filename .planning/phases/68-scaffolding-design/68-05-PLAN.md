**Bard creative brief:**

"Analyze ChatGPT iOS and Claude iOS apps as reference implementations for Loom's native iOS app. Produce a creative direction document covering:

1. **Reference App Analysis** (ChatGPT iOS + Claude iOS):
   - Navigation patterns: drawer behavior, transition animations, gesture areas
   - Message density: spacing between messages, bubble vs flat layout, avatar usage
   - Composer design: height, placeholder text behavior, send button placement
   - Tool/artifact rendering: how they handle code blocks, file references, inline actions
   - Color and surface treatment: background colors, card elevation, border usage
   - Typography: sizes used for different content types, font weights
   - Motion: what animates, spring vs linear, duration ranges
   - What makes them feel 'native' vs 'web-like'

2. **Loom's Elevation Layer** (per D-11):
   - **Motion:** Spring physics configs for every interaction category (button press, screen transition, drawer, card expand, message appear). Include damping + stiffness + mass values.
   - **Depth:** Surface hierarchy (4 levels from UI-SPEC), shadow configs, when to use glass/blur vs solid surface.
   - **Dynamic color:** How color shifts based on conversation context (streaming = subtle pulse, error = warm shift, idle = neutral). Specific animation configs.

3. **Screen-by-Screen Direction** (per D-09):
   For each screen, describe: layout, key components, motion behaviors, color treatment.
   Screens: session list, chat thread, composer, tool cards, permission request, sidebar drawer, settings, share sheet, notification UI, code block detail, search, pinned sessions.

4. **Anti-Patterns** (per quality bar):
   Explicit list of what Loom's native app must NOT do. Examples:
   - No flat/static cards without shadow
   - No instant transitions (always spring)
   - No web-style hover states (use press states)
   - No small touch targets

5. **Elevation Examples:**
   For 3-5 key interactions, describe the exact motion + depth + color response:
   - Sending a message
   - Tool card completing
   - Permission request appearing
   - Drawer opening
   - Error occurring

swd's vision: '$D-12 text'. The web app felt too flat despite 50+ phases. The native app must feel alive from the start.

Baseline tokens are already set in the UI-SPEC (colors, typography, spacing). Build on those, don't replace them unless you have a strong reason."

Write Bard's output to `.planning/phases/68-scaffolding-design/NATIVE-APP-SOUL-DRAFT.md`.

Claude's role: Review Bard's output for completeness against D-09 screen list. If any screens are missing, ask Bard to add them. If spring physics values seem unreasonable (>500ms settle time, damping < 5), flag them. But do NOT override Bard's creative choices -- per D-07, Bard leads creative.
