    Add touch event handlers to the mobile sidebar overlay for swipe-to-close gesture. The sidebar slides in from the left, so a leftward swipe should close it.

    **Implementation approach:**
    1. Add `useRef` for tracking touch state: `touchStartX`, `touchCurrentX`, `isSwiping`
    2. Add a `translateX` state (default 0) that controls the sidebar position during swipe
    3. Wire `onTouchStart`, `onTouchMove`, `onTouchEnd` on the sidebar content wrapper (the `div.h-full.shadow-xl` element)

    **Touch handlers:**
    - `onTouchStart`: Record `e.touches[0].clientX` as start position. Only track if touch starts within the sidebar panel (not the backdrop).
    - `onTouchMove`: Calculate `deltaX = currentX - startX`. If `deltaX < -10` (leftward threshold), set `isSwiping = true`. Apply `translateX` as `Math.min(0, deltaX)` (clamp: only allow leftward movement). Apply via inline `transform: translateX(${px}px)` style for 60fps performance.
    - `onTouchEnd`: If `deltaX < -100` (swipe threshold), call `toggleSidebar()` to close. Otherwise, animate back to `translateX(0)` with a short CSS transition.

    **CSS transition control:**
    - During swipe (isSwiping=true): no transition (direct transform for responsiveness)
    - On release snap-back: add `transition: transform 200ms ease-out` temporarily

    **Key details:**
    - Use `useRef` (not state) for touch tracking to avoid re-renders during gesture
    - Apply transform via ref to the DOM element directly (not React state) for 60fps
    - After close animation, reset all swipe state
    - Keep existing backdrop click and X button close functionality intact
    - The swipe container is the `div.h-full.shadow-xl` wrapping `sidebarContent`

    **Concrete implementation pattern:**
    ```typescript
    const sidebarRef = useRef<HTMLDivElement>(null);
    const touchRef = useRef({ startX: 0, currentX: 0, swiping: false });

    const handleTouchStart = useCallback((e: React.TouchEvent) => {
      touchRef.current.startX = e.touches[0].clientX;
      touchRef.current.swiping = false;
      if (sidebarRef.current) {
        sidebarRef.current.style.transition = 'none';
      }
    }, []);

    const handleTouchMove = useCallback((e: React.TouchEvent) => {
      const deltaX = e.touches[0].clientX - touchRef.current.startX;
      if (deltaX < -10) touchRef.current.swiping = true;
      if (touchRef.current.swiping && sidebarRef.current) {
        const clamped = Math.min(0, deltaX);
        sidebarRef.current.style.transform = `translateX(${clamped}px)`;
      }
    }, []);

    const handleTouchEnd = useCallback(() => {
      const deltaX = touchRef.current.currentX - touchRef.current.startX;
      // ... close or snap back logic
    }, [toggleSidebar]);
    ```

    Wire these handlers on the sidebar panel wrapper div (the one with `className="h-full shadow-xl"`). Pass `ref={sidebarRef}` to that div.

    **A. Viewport meta update (index.html):**
    Add `interactive-widget=resizes-content` to the existing viewport meta tag. This tells mobile browsers to resize the layout viewport when the virtual keyboard appears, which naturally pushes the composer up above the keyboard.

    Change from:
    ```html

    ```
    To:
    ```html
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, interactive-widget=resizes-content" />
    ```

    **D. VisualViewport fallback (ChatComposer.tsx):**
    For browsers that don't fully support `interactive-widget`, add a `useEffect` that listens to `visualViewport.resize` and adjusts the composer position. This is a belt-and-suspenders approach:

    ```typescript
    // Keyboard avoidance fallback for browsers without interactive-widget support
    useEffect(() => {
      if (typeof window === 'undefined') return;
      const vv = window.visualViewport;
      if (!vv) return;

      function handleResize() {
        const vv = window.visualViewport;
        if (!vv) return;
        // When keyboard is open, visualViewport.height < window.innerHeight
        const keyboardHeight = window.innerHeight - vv.height;
        document.documentElement.style.setProperty(
          '--keyboard-offset',
          keyboardHeight > 50 ? `${keyboardHeight}px` : '0px'
        );
      }

      vv.addEventListener('resize', handleResize);
      return () => vv.removeEventListener('resize', handleResize);
    }, []);
    ```

    Then in `base.css`, add:
    ```css
    :root {
      --keyboard-offset: 0px;
    }
    ```

    And in `composer.css`, update the `.composer-safe-area` to also account for keyboard:
    ```css
    @media (max-width: 767px) {
      .composer-safe-area {
        padding-bottom: calc(1rem + var(--keyboard-offset, 0px));
      }
    }
    ```

    Note: The `interactive-widget=resizes-content` approach is the PRIMARY mechanism. The visualViewport listener is a FALLBACK for older browsers. On modern browsers, `interactive-widget` handles it natively without any JS.
