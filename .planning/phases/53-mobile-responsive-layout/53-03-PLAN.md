  <action>
    **A. MessageList padding (MessageList.tsx):**
    The message items wrapper uses `px-4` which is 16px per side = 32px total consumed from the viewport. At 390px, this leaves 358px for content. Reduce padding on mobile for more content room:
    - Change the inner message `div.px-4` (line ~195 and ~203) to `px-2 md:px-4`
    - This gives an extra 16px on mobile (374px available instead of 358px)

    **B. CodeBlock container (CodeBlock.tsx):**
    The code block outer container `div.bg-code-surface.rounded-lg.overflow-hidden.my-3` needs explicit width containment to prevent it from expanding beyond the parent:
    - Add `max-w-full` to the outer container div className
    - The inner `div[data-code-scroll]` already has `overflow-x-auto` which is correct
    - Add `min-w-0` to the outer container to allow flex shrinking

    Also, the code block already handles horizontal scroll via `overflow-x-auto` on the content div. Verify this is present and working (it is, per current code). No additional changes needed for horizontal scroll behavior.

    **C. Images in MarkdownRenderer (MarkdownRenderer.tsx):**
    The img component already has `max-w-full`. Verify this is sufficient. Add `w-auto h-auto` to ensure proportional sizing:
    - Change img className from `"max-w-full rounded-lg cursor-pointer hover:opacity-90 transition-opacity my-2"` to `"max-w-full w-auto h-auto rounded-lg cursor-pointer hover:opacity-90 transition-opacity my-2"`

    **D. ImageThumbnailGrid (ImageThumbnailGrid.tsx):**
    The container uses `flex flex-wrap gap-2`. Images use `max-w-[200px] max-h-[200px]`. On a 374px content width, two 200px images won't fit side by side. Adjust for mobile:
    - Change image className to use responsive max-width: `max-w-[calc(50%-0.25rem)] md:max-w-[200px] max-h-[200px] rounded-lg cursor-pointer object-cover hover:opacity-90 transition-opacity`
    - This ensures images fill half the available width on mobile (with gap), or cap at 200px on desktop

    **E. UserMessage word-break (UserMessage.tsx):**
    The user message content div already has `whitespace-pre-wrap break-words`. Verify this is present.
    - No change needed -- `break-words` is correct and already present.

    **F. Markdown inline code overflow (MarkdownRenderer.tsx):**
    Inline code elements use fixed font-size `text-[0.85em]` and may contain long unbroken strings. Add `break-all` to the inline code className:
    - Change inline code from `"bg-code-inline rounded-sm px-1.5 py-0.5 font-mono text-[0.85em]"` to `"bg-code-inline rounded-sm px-1.5 py-0.5 font-mono text-[0.85em] break-all"`

    **G. Markdown container overflow containment (MarkdownRenderer.tsx):**
    Add `overflow-hidden` to the markdown-body wrapper div to prevent any child from causing horizontal page scroll:
    - Change from `"markdown-body text-foreground text-sm leading-relaxed"` to `"markdown-body text-foreground text-sm leading-relaxed overflow-hidden"`

    Note: Tables already have `overflow-x-auto` wrapper in the MarkdownRenderer table component override (line ~83). No change needed for tables.
