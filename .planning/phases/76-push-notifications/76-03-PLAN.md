**Sync function:**
```typescript
async function syncPreferenceToBackend(newMode: NotificationMode) {
  try {
    const token = getPushToken(); // [S-4] imported from push-preferences.ts (created in Plan 02)
    if (token) {
      await apiClient.patch('/api/push/preferences', { token, notificationMode: newMode });
    }
  } catch (e) {
    console.warn('[Settings] Failed to sync notification preference:', e);
  }
}
```

**[S-4] IMPORTANT:** `getPushToken()` is imported from `mobile/lib/push-preferences.ts` which was created in Plan 02. This plan does NOT add `getPushToken`/`setPushToken` to that file. They already exist. This is what enables Plan 02 and Plan 03 to execute in parallel as wave 1 -- Plan 03 only reads from push-preferences.ts, it does not write to it.
