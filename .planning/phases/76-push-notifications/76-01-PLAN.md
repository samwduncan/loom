**2. Create `server/services/push-service.js`:**

Create a new `server/services/` directory. The PushService is a singleton class:

```javascript
import { Expo } from 'expo-server-sdk';
import { pushTokenDb } from '../database/db.js';
```

**PushService class members:**

- `this.expo = new Expo()` -- Expo Push Service client
- `this.clientStates = new Map()` -- `Map<userId, Array<{ wsId, foreground, viewingSessionId }>>` tracking per-client foreground state
- `this.batchTimers = new Map()` -- `Map<userId, { timer, sessions: [] }>` for 5-second batching per D-04

**Methods:**

- `updateClientState(userId, wsId, foreground, viewingSessionId)` -- upsert client state entry. If entry with wsId exists, update it. If not, push new entry. Remove stale entries when WS disconnects.
- `removeClient(wsId)` -- remove all state entries for this wsId across all userIds
- `isUserViewingSession(userId, sessionId)` -- returns true if ANY client for this user has `foreground === true && viewingSessionId === sessionId` (per D-01)
- `notifySessionComplete(userId, sessionId, sessionName, success, errorMessage)` -- per D-01: skip if `isUserViewingSession`. Per D-16: check `notification_mode` from tokens. Per D-02: 'none' -> skip, 'permissions_only' -> skip, 'failures_and_permissions' && success -> skip. Then queue into batching (D-04).
- `notifyPermissionRequest(userId, sessionId, sessionName, requestId, toolName, description)` -- per D-01: skip if viewing. Per D-16: 'none' -> skip. Permission requests are NEVER batched (time-sensitive). Send immediately.
- `queueSessionComplete(userId, sessionInfo)` -- per D-04: start/reset 5-second timer per userId. On timer fire, call `flushBatch(userId)`.
- `flushBatch(userId)` -- if 1 session: send single notification. If >1: send batched notification "{N} sessions finished". Delete timer entry.
- `async sendPush(tokens, notification)` -- filter with `Expo.isExpoPushToken()`, build messages array per D-21 payload format: `{ to, title, body, data: { type, sessionId, requestId?, url: 'loom://chat/{sessionId}' }, categoryId, sound: 'default', badge: 1 }`. Use `expo.chunkPushNotifications()` and `expo.sendPushNotificationsAsync()`. Log ticket results. Schedule receipt check after 15 seconds.
- `async checkReceipts(tickets)` -- chunk receipt IDs, call `expo.getPushNotificationReceiptsAsync()`. On `DeviceNotRegistered` error: call `pushTokenDb.deleteToken(token)` to remove stale tokens. Log other errors.

**Notification body text per D-21 and UI-SPEC:**
- Session complete success: title = sessionName, body = "Session completed", categoryId = "session_complete"
- Session complete error: title = sessionName, body = "Session failed: {first 80 chars of errorMessage}", categoryId = "session_complete"
- Permission request: title = sessionName, body = "{toolName}: {description, max 80 chars}", categoryId = "permission_request"
- Batched: title = "Sessions completed", body = "{N} sessions finished", categoryId = "session_complete"

Export singleton: `export const pushService = new PushService();`

**1. Create `server/routes/push.js`:**

```javascript
import express from 'express';
import { Expo } from 'expo-server-sdk';
import { pushTokenDb } from '../database/db.js';
const router = express.Router();
```

Endpoints per D-18:

**POST /api/push/register** (register or update token + preferences):
- Body: `{ token: string, platform?: string, notificationMode?: string }`
- Validate `token` exists and `Expo.isExpoPushToken(token)` returns true. If not: 400 `{ error: 'Invalid Expo push token' }`.
- Validate `notificationMode` if provided: must be one of `'all'`, `'failures_and_permissions'`, `'permissions_only'`, `'none'`. Default to `'all'`.
- Call `pushTokenDb.upsertToken(token, platform || 'ios', req.user.id, notificationMode || 'all')`.
- Return 200 `{ success: true }`.

**DELETE /api/push/register** (unregister token):
- Body: `{ token: string }`
- Validate `token` exists. If not: 400.
- Call `pushTokenDb.deleteToken(token)`.
- Return 200 `{ success: true }`.

**PATCH /api/push/preferences** (update notification mode only):
- Body: `{ token: string, notificationMode: string }`
- Validate both fields exist. Validate `notificationMode` is one of the 4 valid values.
- Call `pushTokenDb.updatePreference(token, notificationMode)`.
- Return 200 `{ success: true }`.

All endpoints use try/catch with `res.status(500).json({ error: 'Internal server error' })` on failure. Log errors to console.error.

Export: `export default router;`
