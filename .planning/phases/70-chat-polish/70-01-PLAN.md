     Remove the dynamic FlashList/FlatList require block (lines 33-44 of current file). Replace with:
     ```typescript
     import { LegendList } from '@legendapp/list';
     ```

  2. Replace the ListComponent usage with LegendList. Critical props per research:
     ```tsx
     <LegendList
       data={messages}
       renderItem={renderItem}
       keyExtractor={keyExtractor}
       alignItemsAtEnd
       maintainScrollAtEnd
       maintainVisibleContentPosition
       estimatedItemSize={80}
       contentContainerStyle={styles.contentContainer}
       showsVerticalScrollIndicator={false}
       keyboardShouldPersistTaps="handled"
     />
     ```
     - `alignItemsAtEnd`: Messages align to bottom naturally (no inverted FlatList)
     - `maintainScrollAtEnd`: Auto-scrolls when new items added
     - `maintainVisibleContentPosition`: Preserves scroll position when streaming messages grow
     - `keyboardShouldPersistTaps="handled"`: Prevents Pitfall 11 (two-tap focus bug)

  3. Simplify the auto-scroll logic. Remove the `setInterval` polling for streaming scroll (lines 63-69 of current file) -- `maintainScrollAtEnd` handles this natively. Keep the `useScrollToBottom` hook for the ScrollToBottomPill visibility logic, but remove the `scrollToBottom()` calls from the streaming effect.
