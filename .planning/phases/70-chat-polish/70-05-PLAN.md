  **ToolCallCard.tsx** -- Per Soul doc Tool Cards section:

  1. Create the component with this interface:
     ```typescript
     interface ToolCallCardProps {
       toolName: string;          // "Read", "Write", "Bash", "Search", "Glob", "Grep", "MCP"
       toolInput?: string;        // File path, command, or search pattern
       toolOutput?: string;       // Result content (may be long)
       isRunning: boolean;        // true while tool is executing
       isError: boolean;          // true if tool errored
       isComplete: boolean;       // true when done
       elapsedTime?: string;      // "1.2s", "3.4s"
     }
     ```

  2. Layout per Soul doc:
     - Surface Card (Tier 2, SURFACE.raised, rounded-xl, p-4), inline in message flow
     - **Header row:** Tool icon (16px) + tool name (12px semibold, text-secondary) + status dot (8px) + elapsed time (12px muted)
     - **Collapsed:** Header only, 44px min height. Tappable to expand.
     - **Expanded:** Header + body. Tool input in monospace (14px, surface-sunken inset). Tool output truncated to 4 lines when collapsed, full when expanded.

  3. Status dot colors:
     - Running: ACCENT (pulsing opacity 0.4-1.0, 1.5s cycle)
     - Complete: SUCCESS (static)
     - Error: DESTRUCTIVE (static)

  4. Expand/collapse animation:
     ```tsx
     const [isExpanded, setExpanded] = useState(false);
     const expandHeight = useSharedValue(0);

     const toggleExpand = () => {
       setExpanded(!isExpanded);
       expandHeight.value = withSpring(isExpanded ? 0 : 1, SPRING.expand);
     };
     ```
     Use `Animated.View` with `maxHeight` interpolated from expandHeight for the body content. Content fades in with 100ms delay after expand starts.

  5. Tool icon mapping:
     ```typescript
     const TOOL_ICONS: Record<string, string> = {
       Read: 'doc.text',
       Write: 'doc.text.fill',
       Edit: 'pencil.line',
       Bash: 'terminal',
       Search: 'magnifyingglass',
       Glob: 'folder',
       Grep: 'text.magnifyingglass',
     };
     ```
     Use SF Symbols via expo-symbols, falling back to Lucide icons.

  6. Running state: Pulsing status dot only (opacity 0.4→1.0, 1.5s cycle via Reanimated). NO shimmer animation — shimmer is Phase 71 scope (motion polish). Keep Phase 70 focused on layout and structure.

  **ThinkingBlock.tsx** -- Per Soul doc (expand/collapse disclosure):

  1. Interface:
     ```typescript
     interface ThinkingBlockProps {
       content: string;          // The thinking text
       isStreaming: boolean;     // true while thinking is streaming
       duration?: string;        // "2.3s thinking"
     }
     ```

  2. Layout:
     - No card background (unlike tool cards). Just a disclosure row + content.
     - **Collapsed (default for completed thinking):** Single row: disclosure triangle + "Thinking..." or "Thought for {duration}" in 12px muted italic.
     - **Expanded:** Disclosure row + thinking content rendered as plain text (not markdown) in 14px text-muted, indented 24px from left.
     - During streaming: auto-expanded, thinking content grows as tokens arrive.
     - After streaming: auto-collapses.

  3. Disclosure triangle: Animated rotation (0 = collapsed, 90deg = expanded) using Reanimated.
     ```tsx
     const rotation = useSharedValue(0);
     const toggleExpand = () => {
       const next = !isExpanded;
       setExpanded(next);
       rotation.value = withSpring(next ? 1 : 0, SPRING.expand);
     };
     const triangleStyle = useAnimatedStyle(() => ({
       transform: [{ rotate: `${rotation.value * 90}deg` }],
     }));
     ```

If useStreamStore tracks thinking separately, use that as the isStreaming signal. If thinking is embedded in message content (e.g., `<thinking>...</thinking>` blocks), parse it out and track streaming state locally: set isStreaming=true when content is growing (compare length on each render), false when it stops growing for >500ms. When `isStreaming` transitions from true to false, auto-collapse with Expand spring.

     {message.isStreaming && activeToolCalls.length > 0 && (
