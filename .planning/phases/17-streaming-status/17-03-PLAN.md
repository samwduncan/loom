**useActivityStatus.ts -- new hook:**
- Export `useActivityStatus(claudeStatus, isLoading)` that returns `{ activityText, elapsedSeconds, tokenCount }`.
- `activityText`: Parse from `claudeStatus.text` or tool events. Implement `parseActivityText(toolName, toolInput)` function:
  - "Read" -> `Reading ${basename(file_path)}...`
  - "Write" -> `Writing ${basename(file_path)}...`
  - "Edit" -> `Editing ${basename(file_path)}...`
  - "Bash" -> `Running command...`
  - "Grep" -> `Searching for "${truncate(pattern, 20)}"...`
  - "Glob" -> `Finding files...`
  - "Task" -> `Running subtask...`
  - default -> `Using ${toolName}...`
- `elapsedSeconds`: Start a timer (setInterval 1s) when `isLoading` becomes true, reset to 0 when false.
- `tokenCount`: Use `claudeStatus.tokens` if available, otherwise 0.
- The hook receives the latest tool activity from whatever prop/context the ChatComposer passes down. The tool name/input data is already available via `claudeStatus` or can be threaded from `useChatRealtimeHandlers`.
- Clean up intervals on unmount.

- The stop button in status line calls `onAbort` prop.
