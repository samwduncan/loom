**useScrollAnchor.ts -- full rewrite:**
- Remove IntersectionObserver entirely. Remove sentinelNode state, sentinelCallbackRef, and observer setup useEffect.
- New implementation: listen to scroll events via a `handleScroll` callback that checks `scrollHeight - scrollTop - clientHeight <= 10` (the locked 10px threshold).
- `isAtBottom` state: true when within threshold. Updated on every scroll event.
- `isUserScrolledUp` state: true when NOT at bottom. Auto-resets to false when user scrolls back near bottom (per user decision: auto re-engage, no pill click required).
- `scrollToBottom`: smooth scroll via `container.scrollTo({ top: container.scrollHeight, behavior: 'smooth' })`, resets `isUserScrolledUp` to false.
- Return shape: `{ isAtBottom, isUserScrolledUp, handleScroll, scrollToBottom }`. Note: `sentinelRef` is REMOVED from return, `handleUserScroll` renamed to `handleScroll`.
- Add CSS `overflow-anchor: auto` note in a comment -- the actual CSS will be applied on the container in ChatMessagesPane.
- Do NOT debounce the scroll handler (per research: immediate response required).

**useChatRealtimeHandlers.ts -- tune rAF batching (STRM-01):**
- Current implementation flushes the stream buffer every single rAF (~16ms). Tune to accumulate for 50-100ms.
- Inside the rAF callback (around lines 252-265), add a timestamp check: record `lastFlushTime`. Only flush when `Date.now() - lastFlushTime >= 60` (60ms, middle of 50-100ms range). If not enough time elapsed, re-schedule another rAF without flushing.
- Add a `lastFlushTimeRef = useRef(0)` for the timestamp.
- Keep the existing cleanup logic intact.
