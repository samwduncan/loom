<task type="auto">
  <name>Task 2: Strip all dark: Tailwind prefixes from component files</name>
  <files></files>
  <action>
Programmatically remove all `dark:` Tailwind prefixes from every component file. There are ~1128 occurrences across ~78 files.

**Strategy:** Use `sed` or a script to find-and-replace `dark:` prefixes in className strings. The pattern is: `dark:` followed by a Tailwind utility class (e.g., `dark:bg-gray-800` → `bg-gray-800`, `dark:text-white` → `text-white`, `dark:border-gray-700` → `border-gray-700`).

**IMPORTANT:** After removing `dark:` prefixes, there will be many DUPLICATE classes in the same className string (e.g., `bg-white dark:bg-gray-800` → `bg-white bg-gray-800`). These duplicates are harmless for now — Tailwind applies the last matching class. Phase 2 (Color Sweep) will replace all hardcoded color classes with semantic aliases, which will clean up these duplicates naturally.

**Execution:**

1. Find all files with `dark:` prefixes:
```bash
grep -rl "dark:" src/ --include="*.tsx" --include="*.jsx" --include="*.ts" --include="*.css"
```

2. For each file, remove the `dark:` prefix from Tailwind class names:
```bash
# This regex matches dark: followed by a Tailwind class (letters, numbers, hyphens, brackets, slashes, dots)
# It replaces "dark:bg-gray-800" with "bg-gray-800"
sed -i 's/dark:\([a-zA-Z0-9_\/-]*\(\[[^\]]*\]\)\?[a-zA-Z0-9_\/-]*\)/\1/g' FILE
```

Alternatively, use a simpler approach since `dark:` always prefixes a valid class:
```bash
# Simple: just strip the "dark:" prefix wherever it appears before a class name
find src/ -type f \( -name "*.tsx" -o -name "*.jsx" -o -name "*.ts" -o -name "*.css" \) -exec sed -i 's/dark://g' {} +
```

**CAUTION:** The simple `sed 's/dark://g'` could match `dark:` in strings, comments, or non-Tailwind contexts. Before running, check if `dark:` appears in any non-className context:
```bash
grep -n "dark:" src/ -r --include="*.tsx" --include="*.jsx" | grep -v "className\|class=\|cn(\|clsx(\|tw`" | head -20
```

If there are non-className usages of `dark:`, use a more targeted regex or handle those files manually.

3. Also check `src/index.css` for any remaining `.dark` selectors (should already be clean from Plan 01-01, but verify):
```bash
grep -n "\.dark" src/index.css
```

4. Final verification:
```bash
# Should return 0 matches
grep -rc "dark:" src/ --include="*.tsx" --include="*.jsx" --include="*.ts" --include="*.css" | grep -v ":0$"
```
