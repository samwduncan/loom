<task type="auto">
  <name>Task 1: Configure upstream remote, create upstream-sync branch, and tag fork baseline</name>
  <files></files>
  <action>
Execute the following git operations to establish fork governance:

**1. Rename origin to upstream** (since origin currently points to CloudCLI):
The current `origin` remote points to `https://github.com/siteboon/claudecodeui.git`. This IS the upstream repo. We need to:
- Rename `origin` to `upstream`: `git remote rename origin upstream`
- The Loom repo will eventually get its own remote. For now, upstream is the only remote.

**WAIT — actually, do NOT rename origin.** The user may have push access to origin for their own fork. Instead:
- Check if `upstream` remote already exists: `git remote -v | grep upstream`
- If NOT: `git remote add upstream https://github.com/siteboon/claudecodeui.git`
- If origin IS the CloudCLI repo: origin stays as-is. The user can add their own Loom remote later.

**2. Fetch upstream and create upstream-sync branch:**
```bash
git fetch upstream 2>/dev/null || git fetch origin
git checkout -b upstream-sync HEAD
git checkout main
```

Note: The upstream-sync branch starts at current HEAD (same as main). Future workflow: `git checkout upstream-sync && git fetch upstream && git merge upstream/main` to update the tracking branch, then cherry-pick specific commits to main.

**3. Tag the fork baseline:**
```bash
git tag -a fork-baseline -m "Loom fork baseline — divergence point from CloudCLI"
```

If a tag already exists, skip with a note.

**4. Return to main branch:**
```bash
git checkout main
```
