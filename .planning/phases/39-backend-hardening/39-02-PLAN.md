  <files>server/projects.js, server/index.js, loom-backend.service, ecosystem.config.cjs</files>
  <action>
    1. **updateSessionTitle function** in `server/projects.js`:
       - Add `async function updateSessionTitle(projectName, sessionId, title)` that:
         - Resolves the project directory: `path.join(os.homedir(), '.claude', 'projects', projectName)`
         - Reads all JSONL files to find which file contains the sessionId (same pattern as `deleteSession`)
         - Appends a new line to that JSONL file: `JSON.stringify({ type: 'summary', sessionId, summary: title, timestamp: new Date().toISOString() }) + '\n'`
         - Uses `fs.appendFile` (not read-write-replace) to avoid race conditions
         - Throws if session not found in any file
       - Add `updateSessionTitle` to the module's exports

    2. **PATCH route** in `server/index.js`:
       - Add import of `updateSessionTitle` from `./projects.js`
       - Add route: `app.patch('/api/projects/:projectName/sessions/:sessionId', authenticateToken, async (req, res) => { ... })`
       - Validate `req.body.title` is a non-empty string, max 200 chars
       - Call `updateSessionTitle(projectName, sessionId, title.trim())`
       - Return `{ success: true, title: title.trim() }` on 200
       - Return 400 on missing/invalid title, 404 if session not found, 500 on error
       - Place the route near the existing DELETE session route (line ~534)

    3. **systemd unit file** at project root `loom-backend.service`:
       ```ini
       [Unit]
       Description=Loom AI Backend
       After=network.target

       [Service]
       Type=simple
       User=swd
       WorkingDirectory=/home/swd/loom
       ExecStart=/usr/bin/node server/index.js
       Restart=always
       RestartSec=3
       Environment=NODE_ENV=production
       Environment=PORT=5555
       EnvironmentFile=-/home/swd/loom/.env
       StandardOutput=journal
       StandardError=journal
       SyslogIdentifier=loom-backend

       [Install]
       WantedBy=multi-user.target
       ```
       - The `-` prefix on EnvironmentFile means "don't fail if missing"

    4. **pm2 config** at project root `ecosystem.config.cjs`:
       ```javascript
       module.exports = {
         apps: [{
           name: 'loom-backend',
           script: 'server/index.js',
           cwd: '/home/swd/loom',
           env: { NODE_ENV: 'production', PORT: '5555' },
           restart_delay: 3000,
           max_restarts: 10,
           autorestart: true,
         }],
       };
       ```

    5. **Install and enable systemd service**:
       - `sudo cp loom-backend.service /etc/systemd/system/`
       - `sudo systemctl daemon-reload`
       - `sudo systemctl enable loom-backend`
       - Do NOT start the service yet (user may be running the backend manually during development)
       - Print instructions for `sudo systemctl start loom-backend` and `journalctl -u loom-backend -f`
