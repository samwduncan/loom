    **Internal state architecture (per D-02, D-03):**
    - `isAtBottomRef = useRef(true)` -- hot path, no re-renders
    - `isAutoScrollingRef = useRef(false)` -- anti-oscillation guard
    - `unreadCountRef = useRef(0)` -- hot path unread tracking
    - `[showPill, setShowPill] = useState(false)` -- cold path, debounced 200ms
    - `[displayUnreadCount, setDisplayUnreadCount] = useState(0)` -- cold path, debounced 200ms
    - `debouncedPillUpdate` ref with setTimeout(200ms) that syncs refs to state

    **IntersectionObserver sentinel (per D-02, D-04):**
    - Use callback ref pattern (setSentinelNode state) so observer re-creates when sentinel attaches
    - Observer root: scrollContainerRef.current
    - rootMargin: `'0px 0px -150px 0px'` -- single 150px threshold (Claude's Discretion simplification of D-05's dual 200px/100px hysteresis). RESEARCH.md analysis shows a single threshold combined with the 200ms debounce on schedulePillUpdate() provides equivalent anti-flicker behavior: the debounce window absorbs rapid sentinel enter/exit cycles during programmatic scroll, making the dual-threshold unnecessary. 150px is a middle ground between the show (200px) and hide (100px) values from D-05.
    - On entry.isIntersecting: set isAtBottomRef.current = true, reset unreadCountRef to 0, schedulePillUpdate()
    - On !entry.isIntersecting AND !isAutoScrollingRef.current: set isAtBottomRef.current = false, schedulePillUpdate()
    - Disconnect old observer in useEffect cleanup

    **ResizeObserver auto-follow (per D-06, D-07, D-08):**
    - Observe contentWrapperRef.current (first child of scroll container)
    - Callback: if !isAtBottomRef.current, return. If rafPending, return. Set rafPending=true, rAF(() => { rafPending=false; container.scrollTop = container.scrollHeight - container.clientHeight; })
    - NOT scrollIntoView -- direct scrollTop assignment per D-08
    - Re-setup on session change

    **User gesture detection (port from useScrollAnchor lines 116-152):**
    - wheel + touchmove listeners on container: set isAutoScrollingRef.current = false
    - scroll listener for direction detection: reads `scrollTop` for upward scroll detection only. NOTE (SCROLL-02 compliance): this `scrollTop` read is a non-layout-forcing property read -- it does NOT trigger forced synchronous layout. SCROLL-02's intent is eliminating `scrollHeight - scrollTop - clientHeight` distance calculations that force layout recalculation. A bare `scrollTop` read returns a cached value and is safe in scroll handlers.
    - All listeners passive: true

    **Stream start re-engagement (port from useScrollAnchor lines 72-80):**
    - useEffect watching isStreaming: on false->true edge, set isAtBottomRef.current = true, isAutoScrollingRef.current = true, reset unread

    **Session switch reset:**
    - State-during-render pattern: if sessionId !== prevSessionId, reset isAtBottomRef=true, unreadCountRef=0, isAutoScrollingRef=false, schedulePillUpdate()

    **Unread tracking:**
    - State-during-render pattern: if messageCount > prevMessageCount && !isAtBottomRef.current, increment unreadCountRef by delta, schedulePillUpdate()

    **Scroll position save/restore:**
    - Save: throttled to sessionStorage (200ms trailing) inside the scroll listener (read scrollTop only in this handler, which runs at ~60fps but does NO React state work)
    - Restore: on session switch, read from sessionStorage, apply in rAF

    **statusTap (per D-17, SCROLL-08):**

    - Handler: scrollContainerRef.current?.scrollTo({ top: 0, behavior: 'smooth' })

    **scrollToBottom callback:**
    - container.scrollTo({ top: container.scrollHeight, behavior: 'smooth' })
    - Reset unreadCountRef to 0, isAtBottomRef to true, schedulePillUpdate()
