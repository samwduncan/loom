1. Enhance `src/src/hooks/useScrollAnchor.ts`:

   **Return type expansion** -- add to `UseScrollAnchorReturn`:
   - `unreadCount: number` -- messages arrived while scrolled up
   - `resetUnread: () => void` -- clears unread when user scrolls to bottom
   - `saveScrollPosition: (sessionId: string) => void` -- saves current scrollTop
   - `restoreScrollPosition: (sessionId: string) => void` -- restores saved scrollTop

   **Scroll position map** (NAV-01):
   - Add `scrollPositionMapRef = useRef<Map<string, number>>(new Map())`.
   - `saveScrollPosition`: reads `scrollContainerRef.current.scrollTop`, stores in map.
   - `restoreScrollPosition`: reads from map, sets `scrollContainerRef.current.scrollTop` via `useLayoutEffect`-compatible call (the caller in MessageList will use useLayoutEffect).

   **ResizeObserver bottom lock** (NAV-03):
   - Add a `useEffect` that creates a `ResizeObserver` on the scroll container's first child (content wrapper).
   - In the observer callback: if `isAutoScrollingRef.current` is true, call `sentinelNode?.scrollIntoView({ block: 'end' })` via `requestAnimationFrame` (rAF throttle -- track `rafId` ref, skip if already pending).
   - Clean up observer and cancel pending rAF on unmount.

   **Unread count** (NAV-04):
   - Add `unreadCountRef = useRef(0)` and `[unreadCount, setUnreadCount] = useState(0)`.
   - Expose a method or use an effect: when `isAtBottom` is false and the sentinel triggers (new content appeared), increment unread count.
   - Actually, track unread in the IntersectionObserver callback: when NOT intersecting and content grows, we can't easily detect "new messages". Better approach: expose `incrementUnread()` from the hook, and have MessageList call it when a new message arrives while `showPill` is true.
   - When user scrolls to bottom (clicks pill or auto-scroll kicks in), reset unread to 0.
   - Return `unreadCount` and `resetUnread`.

   **Pill threshold** (NAV-04):
   - The existing pill uses IntersectionObserver on sentinel. The 200px threshold should be handled via `rootMargin: '0px 0px -200px 0px'` or by checking scrollTop distance. The current implementation may already approximate this via the sentinel approach. If the existing logic uses intersection ratio, update the IntersectionObserver rootMargin to `-200px` on the bottom so the pill only appears when the sentinel is 200px+ out of view.

   - Add `unreadCount?: number` prop to `ScrollToBottomPillProps`.
   - When `unreadCount > 0`, render a small circular badge in the top-right corner of the pill: `<span className="scroll-pill-badge">{unreadCount}</span>`.
   - Badge styling: `absolute -top-1.5 -right-1.5 min-w-[1.25rem] h-5 rounded-full bg-primary text-primary-foreground text-xs flex items-center justify-center font-medium`.
