  <action>
Create `src/components/chat/styles/` directory (if it doesn't exist).

**aurora-shimmer.css:**
Register CSS custom property `--aurora-angle` with `@property` (syntax: `<angle>`, initial-value: `0deg`, inherits: false).

Define `@keyframes aurora-rotate` that animates `--aurora-angle` from `0deg` to `360deg`.

Create these utility classes:
- `.aurora-gradient` — base class with `linear-gradient(calc(var(--aurora-angle) + 45deg), ...)` using oklch color stops across the full rainbow spectrum (red, orange, yellow, green, cyan, blue, purple, back to red). Use approximately: `oklch(65% 0.28 0)` (red), `oklch(70% 0.25 45)` (orange), `oklch(75% 0.22 90)` (yellow), `oklch(70% 0.25 150)` (green), `oklch(65% 0.28 210)` (cyan), `oklch(60% 0.28 270)` (blue), `oklch(65% 0.25 330)` (purple), `oklch(65% 0.28 360)` (red). Animation: `aurora-rotate 4s linear infinite`.
- `.aurora-skeleton-line` — extends aurora gradient behavior. Height: 14px, border-radius: 4px, margin-bottom: 8px, opacity: 0.5. Must include the full gradient + animation (not CSS `composes` — just repeat the gradient).
- `.aurora-thinking-text` — the gradient with brighter oklch values (L: 75-85%), plus `-webkit-background-clip: text`, `background-clip: text`, `color: transparent`, `font-weight: 500`. Animation: same aurora-rotate 4s.

**streaming-cursor.css:**
Define `@keyframes cursor-blink` — opacity 1 at 0%/100%, opacity 0 at 50%.
Define `.streaming-cursor::after` — content: "|", color: `#d4a574` (amber), font-weight: bold, animation: `cursor-blink 1s step-end infinite`, margin-left: 1px.
Define `.message-streaming` — `contain: content` (shorthand for layout + paint + style containment). This prevents layout changes inside streaming messages from triggering sibling/parent recalculation.

**Import in main.tsx:**
Add CSS imports at the top of `src/main.tsx`:
```typescript
import './components/chat/styles/aurora-shimmer.css';
import './components/chat/styles/streaming-cursor.css';
```
Place these after existing CSS imports but before component imports.

<task type="auto">
  <name>Task 2: Wire streaming cursor into Markdown component</name>
  <files>src/components/chat/view/subcomponents/Markdown.tsx</files>
  <action>
In `Markdown.tsx`, modify the component to add a streaming cursor when `isStreaming` is true.

The approach: Wrap the ReactMarkdown output in a container `<div>`. When `isStreaming` is true, add the class `streaming-cursor` to this container div. The `::after` pseudo-element defined in `streaming-cursor.css` will append the blinking amber caret at the end of the rendered content.

Import the streaming-cursor CSS file at the top of Markdown.tsx:
```typescript
import '../../styles/streaming-cursor.css';
```

In the JSX return, wrap the `<ReactMarkdown>` in a `<div>`:
```tsx
return (
  <div className={`${isStreaming ? 'streaming-cursor message-streaming' : ''}`}>
    <ReactMarkdown ... />
  </div>
);
```

The `message-streaming` class applies `contain: content` during streaming for layout isolation. Both classes are removed when streaming completes (isStreaming becomes false), so the caret disappears and containment is lifted.

Note: The CSS import for `streaming-cursor.css` is technically already global via main.tsx, but importing it here too is fine for colocation and ensures the dependency is clear.
