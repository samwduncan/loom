Create `src/components/chat/hooks/useScrollAnchor.ts`:

```typescript
import { useCallback, useEffect, useRef, useState } from 'react';
```

**Hook signature:**
```typescript
export function useScrollAnchor(scrollContainerRef: React.RefObject<HTMLDivElement>)
```

**Internal state:**
- `sentinelRef` — useRef for the sentinel div element
- `isAtBottom` state (boolean, default true) — driven by IntersectionObserver
- `isAtBottomRef` — synchronous ref mirror of isAtBottom (for event handler reads)
- `isUserScrolledUp` state (boolean, default false) — set true on user scroll-away, set false when sentinel becomes visible again

**IntersectionObserver setup (useEffect):**
- Create observer with options: `{ threshold: 0, rootMargin: '100px' }` (100px tolerance from CONTEXT.md)
- On intersection callback:
  - If `entry.isIntersecting` → set `isAtBottom` true, `isAtBottomRef` true, set `isUserScrolledUp` false (auto re-engage per user decision)
  - If not intersecting → set `isAtBottom` false, `isAtBottomRef` false
- Observe `sentinelRef.current`
- Cleanup: disconnect observer

**handleUserScroll (useCallback):**
- Called on `onWheel`/`onTouchMove` events on the scroll container
- If `!isAtBottomRef.current` → set `isUserScrolledUp` true (immediate pause, no debounce per user decision)

**scrollToBottom (useCallback):**
- If `scrollContainerRef.current` exists, set `scrollContainerRef.current.scrollTo({ top: scrollContainerRef.current.scrollHeight, behavior: 'smooth' })` — smooth scroll (~300ms per user decision)
- After calling scrollTo, immediately set `isUserScrolledUp` false (optimistic update)

**sentinelCallbackRef (useCallback):**
- A ref callback function for the sentinel div element that sets `sentinelRef.current`

**Return:**
```typescript
return {
  sentinelRef: sentinelCallbackRef,
  isAtBottom,
  isUserScrolledUp,
  handleUserScroll,
  scrollToBottom,
};
```

**Important implementation notes:**
- Use `isAtBottomRef` (synchronous ref) for decisions in `handleUserScroll` — IntersectionObserver callbacks are async and may lag behind React state
- The sentinel div will be placed at the bottom of the message list by ChatMessagesPane (Plan 07-03)
- `overflow-anchor: auto` CSS will be applied by ChatMessagesPane to the scroll container

```typescript
import { useCallback, useEffect, useRef, useState } from 'react';
```

**Hook signature:**
```typescript
export function useNewTurnCounter(turnCount: number, isUserScrolledUp: boolean)
```

**Logic:**
- `snapshotRef` — useRef<number> storing the turnCount at the moment user scrolled up
- `newTurnCount` state (number, default 0)

**Effect (depends on isUserScrolledUp, turnCount):**
- When `isUserScrolledUp` becomes true and `snapshotRef.current === null` → snapshot: `snapshotRef.current = turnCount`
- When `isUserScrolledUp` becomes false → reset: `snapshotRef.current = null`, set `newTurnCount` to 0
- When `isUserScrolledUp` is true → update: set `newTurnCount` to `Math.max(0, turnCount - (snapshotRef.current ?? turnCount))`

Use `null` as the initial/reset value for snapshotRef to distinguish "not scrolled up" from "scrolled up at turn 0".

**Return:**
```typescript
return { newTurnCount };
```
