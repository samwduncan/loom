Actually, `isVisible` is already true when this renders (because it's inside `{isLoading && ...}`). But for the collapse transition to work properly, we need the component to stay mounted briefly after isLoading becomes false. Change the approach:

Track the previous isLoading state to allow collapse animation:
```typescript
const [showIndicator, setShowIndicator] = useState(false);
const indicatorTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

useEffect(() => {
  if (isLoading) {
    setShowIndicator(true);
    if (indicatorTimerRef.current) {
      clearTimeout(indicatorTimerRef.current);
      indicatorTimerRef.current = null;
    }
  } else if (showIndicator) {
    // Keep mounted for collapse animation
    indicatorTimerRef.current = setTimeout(() => {
      setShowIndicator(false);
    }, 350); // Slightly longer than 300ms collapse
  }
  return () => {
    if (indicatorTimerRef.current) clearTimeout(indicatorTimerRef.current);
  };
}, [isLoading]);
```

Then render:
```tsx
{showIndicator && <AssistantThinkingIndicator selectedProvider={provider} isVisible={isLoading} />}
```

When `isLoading` becomes false, `isVisible` goes false → PreTokenIndicator collapses upward over 300ms → then `showIndicator` becomes false → unmount.
