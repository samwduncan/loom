<task type="auto">
  <name>Task 2: Fix global transition reset and clean up redundant overrides</name>
  <files>src/index.css</files>
  <action>
Three changes to the transition system in index.css:

**1. Remove `transition: none` from the global `*` rule (line 107):**
The `* {}` block in @layer base should become:
```css
* {
  border-color: hsl(var(--border) / 0.08);
  box-sizing: border-box;
  /* transition: none -- REMOVED per DSGN-15 */
}
```
Note: Also update the border opacity from 0.15 to 0.08 here (per DSGN-11 -- subtle white borders). This is a single-line change done alongside the transition removal since they share the same `*` rule.

**2. Remove redundant manual transition override classes (lines 168-190):**
Delete these entire rule blocks from @layer base:
- `.transition-all { transition: all 150ms ... }` (lines 168-170)
- `.transition-transform { transition: transform 150ms ... }` (lines 173-175)
- `.transition-opacity { transition: opacity 200ms ... }` (lines 178-180)
- `.transition-scale { transition: transform 100ms ... }` (lines 183-185)
- `.transition-shadow { transition: box-shadow 200ms ... }` (lines 188-190)

These were workarounds for the `transition: none` reset. With the reset gone, Tailwind's native utility classes handle these.

**3. Simplify the interactive element transition defaults (lines 162-170):**
Currently the selector includes `.transition-all` (being removed). Change to:
```css
button,
a,
input,
textarea,
select,
[role="button"] {
  transition-property: color, background-color, border-color, opacity, box-shadow;
  transition-duration: 150ms;
  transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
}
```
This replaces `transition: all 150ms` with specific properties to avoid transitioning width/height/transform by default (per research Pitfall 3). Remove `.transition-all` from the selector list.

**DO NOT touch:**
- `.sidebar-transition`, `.modal-transition`, `.message-transition`, `.height-transition` -- these encode specific multi-property transitions
- The `@media (prefers-reduced-motion: reduce)` block
- The hover/active/focus-visible duration overrides in @layer utilities
