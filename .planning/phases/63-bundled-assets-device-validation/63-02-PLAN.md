<objective>
Add native-aware connection error handling and create the device validation checklist.

Purpose: When the app is bundled as a Capacitor iOS app, the server (100.86.4.57:5555) is only reachable via Tailscale VPN. If the device isn't on VPN, `bootstrapAuth()` throws an unhandled error, leaving the user with a blank screen after the splash hides. This plan adds (a) error handling in websocket-init.ts to catch auth failure and set a meaningful error state, (b) native-specific messaging in ConnectionBanner, and (c) a comprehensive device validation checklist for on-device testing.

Output: Enhanced ConnectionBanner with IS_NATIVE-aware messages, websocket-init.ts with auth error handling, updated tests, and DEVICE-VALIDATION-CHECKLIST.md.
</objective>

    **websocket-init.ts changes (D-09):**

    1. Add import at top: `import { IS_NATIVE } from '@/lib/platform';`

    2. Replace lines 349-351 (the bare `const token = await bootstrapAuth(); wsClient.connect(token);`) with a try/catch block:
    ```typescript
    // Bootstrap auth and connect
    try {
      const token = await bootstrapAuth();
      wsClient.connect(token);
    } catch (err: unknown) {
      const message = IS_NATIVE
        ? 'Server unreachable \u2014 check Tailscale VPN connection'
        : 'Unable to connect to server';
      connectionStore().setProviderError('claude', message);
      connectionStore().updateProviderStatus('claude', 'disconnected');
    }
    ```

    This ensures that when the server is unreachable (common on iOS when not on VPN), the connection store gets an error message that ConnectionBanner will display after the splash screen hides.
