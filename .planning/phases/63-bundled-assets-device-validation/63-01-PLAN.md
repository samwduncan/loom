  <action>
    1. In `src/vite.config.ts`, add `base: './',` as the first property inside defineConfig (before `plugins`). This makes Vite output relative asset paths (`./assets/index-xxx.js` instead of `/assets/index-xxx.js`) and rewrite CSS `url()` references to relative paths (`./fonts/inter-variable.woff2` instead of `/fonts/inter-variable.woff2`). Per D-06.

    2. In `src/index.html`, add `style="background-color:#2b2521"` to the `<html>` tag (making it `<html lang="en" style="background-color:#2b2521">`). Per D-12 — this hex value matches the SplashScreen backgroundColor in capacitor.config.ts and is visually equivalent to the app's oklch(0.20 0.010 32) surface color.

    3. In `src/index.html`, add `style="background-color:#2b2521"` to the `<body>` tag (making it `<body style="background-color:#2b2521">`). Per D-12 — the CSS `background-color: var(--surface-base)` will override this once loaded, but the inline style prevents the white flash gap between splash dismissal and first CSS paint.

    IMPORTANT: Do NOT change the `<script>` tag's `src="/src/main.tsx"` — Vite's dev server needs the absolute path for HMR. The `base` option only affects the production build output, not the dev server source references.
  </action>
